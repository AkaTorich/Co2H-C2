Генератор артефактов

Артефакт — это шаблон beacon'а с уже встроенными адресом C2 и ключом listener'а. Поддерживаются две целевые ОС и четыре транспорта.

Шаблоны по целевой ОС

Все шаблоны лежат в каталоге bin\beacons\ рядом с клиентом и artifact-gen. Выбор шаблона полностью автоматический — достаточно указать целевую ОС, формат и архитектуру в диалоге генерации.

Итого 16 шаблонов для трёх ОС. Все доступны через диалог генерации артефакта в клиенте (Toolbar → Generate artifact): выбор ОС автоматически обновляет списки форматов и архитектур.

Генерация полностью интегрирована в клиент — отдельные команды или утилиты запускать не нужно. Подробнее о диалоге: Диалог генерации артефакта.

Создание из клиента

Toolbar → Generate artifact:

  1. В dropdown Listener выбрать нужный listener из списка — ключ и параметры подставляются автоматически.
  2. Поля меняются по типу listener'а:
    • HTTPS — C2 host и port.
    • TCP — C2 host и port; beacon выберет TCP-транспорт автоматически.
    • SMB — имя pipe (берётся из listener'а автоматически) и Server host — IP или hostname машины, к pipe которого подключится beacon.
    • DNS — поле C2 domain (заполняется из listener'а автоматически). Поле C2 host скрыто — IP teamserver'а подставляется автоматически из адреса текущего подключения.
  3. (Необязательно) В поле Fallback C2 указать резервные серверы (до 4, по одному на строку, формат host:port[:uri]). См. Fallback C2.
  4. В поле Target OS выбрать Windows, Linux или macOS.
  5. Выбрать формат и архитектуру:
    • Windows — EXE / DLL, x64 / x86.
    • Linux — ELF / SO, x64 / ARM64 / ARM32.
    • macOS — Mach-O / DYLIB, Universal (arm64 + x64) / ARM64 / x64.
  6. Указать путь сохранения и нажать OK. Прогресс и сообщения отображаются во вкладке Logs.

Генерация relay child beacon

Контекстное меню beacon'а (Sessions или Graph) → Generate relay child beacon… открывает диалог генерации дочернего артефакта, настроенного подключаться через выбранный родительский beacon.

ПолеОписание
Target OSЦелевая ОС дочернего beacon'а: Windows или Linux.
FormatEXE или DLL (только для Windows; при выборе Linux — скрыто).
Architecturex64 или x86 (только для Windows; при выборе Linux — скрыто).
OutputПуть сохранения файла. При выборе Linux расширение .exe/.dll автоматически убирается.

Артефакт генерируется с параметрами:

  • Host — IP родительского beacon'а (внутренний адрес цели).
  • Port — relay-порт, открытый на родительском beacon'е.
  • Parent ID — hex-идентификатор родительского beacon'а (патчится в parent_id структуры BeaconState).
  • Транспорт — raw TCP (tcp:// в uri_checkin).
Перед генерацией дочернего артефакта необходимо открыть relay-порт на родительском beacon'е: контекстное меню → Start relay port (chain pivot)…. Дочерний beacon подключается к этому порту и получает доступ к teamserver'у через цепочку ретрансляции.

Требования для DNS-артефакта

Для работы DNS C2 необходимо:

  1. Делегировать поддомен (например c2.example.com) на IP teamserver'а через NS-запись у регистратора.
  2. Открыть UDP-порт 53 на файрволе teamserver'а.
  3. Создать DNS listener через New listener → DNS (UDP), указав тот же поддомен в поле C2 domain.
  4. Сгенерировать артефакт, выбрав этот DNS listener. Поле C2 domain заполнится автоматически.
