Быстрый старт

1. Сертификаты mTLS

Генерируются автоматически при первой сборке проекта или вручную через меню Tools → Generate Certificates. Файлы появляются в каталоге certs\: ca.crt, server.crt/server.key, operator.crt/operator.key.

2. Запуск клиента

Двойной клик по co2h-client.exe. В диалоге логина:

ПолеЗначение по умолчанию
Host127.0.0.1
Port5050
Usernameadmin
Passwordchangeme
CAcerts\ca.crt
Cert / Keycerts\operator.crt / certs\operator.key

3. Запуск teamserver'а

На toolbar'е клиента — кнопка Start Server. Запускает встроенный teamserver рядом с клиентом, использует configs/srv.toml. Stdout/stderr процесса идут во вкладку Logs. Stop Server завершает его.

4. Создание listener'а

HTTPS-listener запускается автоматически из srv.toml. TCP, SMB и DNS создаются через клиент: toolbar → New listener (или кнопка Add во вкладке Listeners), выбрать тип, указать параметры. После создания listener появится в списке с ключом.

Для DNS: указать C2 domain — поддомен, авторитарным сервером для которого является teamserver (NS-запись в DNS регистратора).

5. Генерация beacon'а

Toolbar → Generate artifact. В dropdown Listener выбрать нужный listener — ключ и параметры подставятся автоматически. Указать формат (EXE/DLL), sleep/jitter и путь сохранения.

Beacon зашивает адрес и ключ выбранного listener'а. При запуске он подключается именно к этому listener'у; teamserver идентифицирует beacon по зашифрованному check-in.

6. Запуск beacon'а на цели

Скопировать payload.exe на тестовую машину (Windows 10/11 x64) и запустить. Через несколько секунд beacon появится во вкладке Sessions.

7. Взаимодействие

Двойной клик по строке beacon'а → нижняя консоль станет активной. Введите pwd и нажмите Enter.
