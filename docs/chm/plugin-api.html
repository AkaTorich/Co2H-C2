Plugin API

Клиент Co2H поддерживает динамические плагины — разделяемые библиотеки (.dll на Windows, .so на Linux), которые загружаются из папки plugins/ рядом с исполняемым файлом co2h-client при запуске.

Плагин может:

  • Добавлять вкладки в главное окно
  • Регистрировать консольные команды (с автодополнением и подсказками)
  • Отправлять задачи beacon'ам и получать результат через callback
  • Писать в консоль и журнал (Logs)
  • Читать модели данных (сессии, listener'ы, учётные данные, загрузки)

Структура плагина

Каждый плагин — отдельная DLL/SO, экспортирующая две C-функции:

#include "client/plugin/IClientPlugin.hpp"

CO2H_PLUGIN_EXPORT
co2h::client::plugin::IClientPlugin* co2h_plugin_create();

CO2H_PLUGIN_EXPORT
void co2h_plugin_destroy(co2h::client::plugin::IClientPlugin*);

Макрос CO2H_PLUGIN_EXPORT раскрывается в extern "C" __declspec(dllexport) на Windows и extern "C" __attribute__((visibility("default"))) на Linux.

Интерфейс IClientPlugin

namespace co2h::client::plugin {

struct PluginInfo {
    const char* name;        // короткое имя ("my_recon")
    const char* version;     // "1.0.0"
    const char* author;
    const char* description; // одна строка
};

class IClientPlugin {
public:
    virtual ~IClientPlugin() = default;

    // Метаданные плагина.
    virtual PluginInfo info() const = 0;

    // Вызывается один раз после загрузки.
    // ctx — объект PluginContext для взаимодействия с клиентом.
    virtual void initialize(PluginContext* ctx) = 0;

    // Вызывается перед выгрузкой. Освободить ресурсы.
    virtual void shutdown() = 0;
};

}

PluginContext — API для плагинов

Управление виджетами плагина

МетодОписание
int addPluginButton(QWidget* w, const QIcon& icon, const QString& label) Добавить виджет плагина. На панели плагинов появится иконка-кнопка. Первый клик по кнопке открывает интерфейс плагина (основные вкладки скрываются), повторный клик по той же кнопке скрывает плагин и возвращает вкладки клиента. Возвращает индекс. Виджет принадлежит плагину — удалять его нужно в shutdown().
void removePluginButton(QWidget* w) Убрать ранее добавленный виджет плагина.

Выполнение задач

Именованные методы для типовых операций — плагину не нужно знать числовые коды протокола:

МетодОписание
shell(beaconId, cmd, cb) Выполнить команду через cmd.exe /c.
run(beaconId, cmdline, cb) Запустить процесс напрямую (без cmd.exe).
ps(beaconId, cb) Список процессов.
ls(beaconId, path, cb) Содержимое каталога (пустой путь = текущий).
pwd(beaconId, cb) Текущий каталог.
getuid(beaconId, cb) Текущий пользователь / SID.
screenshot(beaconId, cb) Снимок экрана.
download(beaconId, remotePath, cb) Скачать файл с цели.
upload(beaconId, payload, cb) Загрузить данные на цель.

Каждый метод возвращает rpc_id (0 при ошибке). cb — необязательный callback void(const QByteArray& output, const QString& error), вызывается при получении результата.

Маршрутизация полностью автоматическая: rpc_id привязывается к callback, при ответе сервера привязка переносится на task_id, при поступлении вывода — callback вызывается и удаляется.

Для нестандартных операций (BOF, пользовательские расширения) доступен низкоуровневый метод sendTask(beaconId, op, payload, cb), принимающий числовой код операции.

Выпол��ение встроенных команд

МетодОписание
execCommand(beaconId, command, cb) Выполнить любую встроенную команду клиента по имени и получить вывод в callback. Строка command — полная команда как в консоли, например "bof enum.o go -- z domain.local" или "hashdump". Можно вызывать несколько раз подряд — каждый callback получит свой результат.

Пример использования:

ctx_->execCommand(bid, "hashdump",
    [this](const QByteArray& out, const QString& err) {
        if (err.isEmpty())
            parseHashes(QString::fromUtf8(out));
        else
            ctx_->consoleError("[myplugin] hashdump failed: " + err);
    });

Пример: плагин «Recon»

Минимальный плагин, который добавляет вкладку Recon и консольную команду recon. При вызове запускает три разведывательные команды на beacon'е и собирает результат в свою вкладку.

#include "client/plugin/IClientPlugin.hpp"
#include "client/plugin/PluginContext.hpp"

#include <QTextEdit>
#include <QVBoxLayout>
#include <QWidget>

using namespace co2h::client;
using namespace co2h::client::plugin;

class ReconPlugin : public IClientPlugin {
public:
    PluginInfo info() const override {
        return {"recon", "1.0.0", "operator",
                "Basic recon: whoami + ipconfig + net user"};
    }

    void initialize(PluginContext* ctx) override {
        ctx_ = ctx;

        // Создаём виджет вкладки.
        tab_ = new QWidget;
        out_ = new QTextEdit(tab_);
        out_->setReadOnly(true);
        auto* lay = new QVBoxLayout(tab_);
        lay->addWidget(out_);

        ctx_->addPluginButton(tab_, QIcon(), "Recon");

        // Регистрируем команду.
        ctx_->registerCommand(
            "recon", "[target]",
            "whoami + ipconfig + net user на beacon'е",
            [this](const QString& bid, const QString&) {
                runRecon(bid);
            });
    }

    void shutdown() override {
        if (ctx_ && tab_) ctx_->removePluginButton(tab_);
        delete tab_; tab_ = nullptr;
    }

private:
    void runRecon(const QString& bid) {
        ctx_->consoleWrite("[recon] starting...");

        ctx_->shell(bid, "whoami /all",
            [this](const QByteArray& o, const QString& e) {
                if (e.isEmpty())
                    out_->append("=== whoami ===\n"
                        + QString::fromLocal8Bit(o));
                else
                    out_->append("ERROR: " + e);
            });

        ctx_->shell(bid, "ipconfig /all",
            [this](const QByteArray& o, const QString& e) {
                if (e.isEmpty())
                    out_->append("=== ipconfig ===\n"
                        + QString::fromLocal8Bit(o));
            });
    }

    PluginContext* ctx_ = nullptr;
    QWidget*       tab_ = nullptr;
    QTextEdit*     out_ = nullptr;
};

// Экспортируемые функции.
CO2H_PLUGIN_EXPORT IClientPlugin* co2h_plugin_create() {
    return new ReconPlugin;
}
CO2H_PLUGIN_EXPORT void co2h_plugin_destroy(IClientPlugin* p) {
    delete p;
}

Иконка плагина

Каждый плагин хранит свою иконку внутри исходного кода как строковую константу с полным текстом SVG-файла. Иконка не зависит от ресурсов клиента — при загрузке плагин сам рендерит SVG в QIcon.

Шаблон встраивания иконки

#include <QIcon>
#include <QPainter>
#include <QPixmap>
#include <QSvgRenderer>

// Полный текст SVG — хранится как raw string literal.
static const char kMyPluginSvg[] = R"SVG(
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24">
  <!-- ... содержимое иконки ... -->
</svg>
)SVG";

// Вспомогательная функция: SVG-строка → QIcon.
static QIcon iconFromSvg(const char* data) {
    QByteArray svg(data);
    QSvgRenderer r(svg);
    QPixmap pix(24, 24);
    pix.fill(Qt::transparent);
    QPainter p(&pix);
    r.render(&p);
    return QIcon(pix);
}

Использование при регистрации

void initialize(PluginContext* ctx) override {
    // ...
    ctx->addPluginButton(tab_, iconFromSvg(kMyPluginSvg), "My Plugin");
}
Требования к SVG:
  • Размер viewBox — 0 0 24 24 (стандартный для иконок клиента).
  • Допускаются градиенты, opacity, любые SVG-примитивы.
  • Не используйте внешние ссылки (xlink:href на внешние файлы).
  • Для рендеринга необходим модуль Qt6::Svg — добавьте его в CMakeLists.txt (см. ниже).

Сборка плагина

Минимальный CMakeLists.txt для плагина:

cmake_minimum_required(VERSION 3.24)
project(my_plugin LANGUAGES CXX)

find_package(Qt6 6.4 COMPONENTS Core Gui Widgets Svg REQUIRED)
set(CMAKE_AUTOMOC ON)

add_library(my_plugin SHARED MyPlugin.cpp)
target_link_libraries(my_plugin PRIVATE
    Qt6::Core Qt6::Gui Qt6::Widgets Qt6::Svg)

set_target_properties(my_plugin PROPERTIES
    OUTPUT_NAME "my_plugin"
    PREFIX ""
    RUNTIME_OUTPUT_DIRECTORY "${CMAKE_BINARY_DIR}/plugins"
)
Плагин должен быть собран тем же компилятором и версией Qt, что и клиент. Несовпадение приведёт к крашу при загрузке.

Жизненный цикл

  1. При запуске клиент сканирует plugins/ на наличие *.dll (Windows) или *.so (Linux).
  2. Для каждого файла вызывается LoadLibrary / dlopen.
  3. Ищутся экспорты co2h_plugin_create и co2h_plugin_destroy. Если хотя бы один не найден — файл пропускается.
  4. Вызывается co2h_plugin_create() → указатель на IClientPlugin.
  5. Вызывается initialize(ctx) — плагин добавляет вкладки, команды и т.д.
  6. При завершении клиента: shutdown() → co2h_plugin_destroy(ptr) → FreeLibrary / dlclose.

Количество загруженных плагинов записывается во вкладку Logs при старте.

Рекомендации

  • Удаляйте свои виджеты в shutdown(), а не в деструкторе — к моменту вызова деструктора QTabWidget может быть уже уничтожен.
  • Callback из sendTask вызывается один раз. Для повторяющихся задач отправляйте новый sendTask из callback'а.
  • Не блокируйте UI-поток длительными операциями — используйте QThread или QtConcurrent.
  • Модели (sessionsModel() и т.д.) — только для чтения. Запись через них не запрещена, но не рекомендуется.
  • Для отправки задач используйте числовые opcode'ы из beacon.h. Новые opcode'ы можно определять совместно с кастомными BOF.
Кодировка вывода beacon'а. Beacon отправляет текст в UTF-8 на Windows и в системной кодовой странице на Linux. При декодировании QByteArray из callback'а используйте условную компиляцию: 
#ifdef _WIN32
    QString text = QString::fromUtf8(output);
#else
    QString text = QString::fromLocal8Bit(output);
#endif
Без этого русские символы будут отображаться некорректно.
