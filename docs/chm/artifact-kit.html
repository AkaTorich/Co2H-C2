Artifact Kit

Artifact Kit — набор шаблонных исполняемых файлов (стабов), в которые artifact-gen встраивает скомпилированный beacon DLL. Каждый стаб реализует свою технику доставки/загрузки бикона на цель. Все стабы компилируются без CRT (/NODEFAULTLIB, /Zl, /GS-).

Структура каталога

kit/artifact/
+-- artifact.h            -- общий заголовок (макросы, хелперы)
+-- build_all.bat         -- сборка всех стабов (x64 + x86)
+-- exe_basic/            -- базовый EXE: temp file + LoadLibraryW
+-- exe_debug/            -- отладочный EXE: MessageBox на каждом этапе
+-- exe_svc/              -- Windows Service
+-- exe_reflective/       -- In-memory PE loader (без записи на диск)
+-- exe_inject/           -- Remote DLL injection в другой процесс
+-- exe_ppid/             -- PPID-spoofing + injection
+-- dll_sideload/         -- DLL для side-loading / search-order hijack
+-- artifact/             -- выходной каталог (скомпилированные .exe/.dll)

Секция .co2pay

Каждый стаб содержит именованную PE-секцию .co2pay — слот для beacon DLL. Сентинел при компиляции:

43 4F 32 48 50 41 59 4C  00 00 00 00 ...   ("CO2HPAYL" + 4-byte size + zeros)

artifact-gen находит этот сентинел в скомпилированном стабе, записывает размер DLL в байты [8..11] и копирует тело DLL начиная со смещения 12. Максимальный размер: 512 КБ.

Макрос из artifact.h:

ART_DECLARE_PAYLOAD();
// Раскрывается в:
//   #pragma section(".co2pay", read)
//   __declspec(allocate(".co2pay"))
//   static const unsigned char g_payload[512*1024 + 12] = {"CO2HPAYL", 0, ...};

Общий заголовок artifact.h

Файл kit/artifact/artifact.h содержит повторяющийся код, общий для всех стабов:

Функция / макросНазначение
ART_DECLARE_PAYLOAD()Объявление секции .co2pay с сентинелом
art_get_size(payload)Проверка магика + возврат размера DLL (0 при ошибке)
art_memcpy(dst, src, n)Побайтовое копирование без CRT
art_write_temp(path, payload, sz)Запись DLL во временный файл (%TEMP%)
art_load_beacon(payload, sz)Запись + LoadLibraryW + удаление файла
art_inject_remote(hProcess, path)Инжект через CreateRemoteThread(LoadLibraryW)

Все функции объявлены как static — каждый стаб получает собственную копию; неиспользуемые удаляются линкером.

Описание стабов

exe_basic

Минимальный EXE. Записывает DLL во временный файл, загружает через LoadLibraryW, удаляет файл, засыпает навсегда. Точка входа: stub_main. Используйте как шаблон для собственных стабов.

exe_debug

Идентичен exe_basic, но показывает MessageBoxA на каждом этапе и устанавливает Vectored Exception Handler (VEH) для отлова исключений внутри DllEntry beacon'а. Используйте при отладке проблем с загрузкой.

Не используйте в операциях — диалоги видны пользователю.

exe_svc

Обёртка Windows Service. При запуске через SCM регистрирует ServiceCtrlHandler, сообщает SERVICE_RUNNING и держит beacon живым. При прямом запуске (не от SCM) молча переключается на поведение exe_basic.

sc create BeaconSvc binPath= "C:\path\exe_svc.x64.exe" start= auto
sc start  BeaconSvc
Service API (advapi32.dll) загружается динамически — стаб линкуется только с kernel32.lib + user32.lib.

exe_reflective

In-memory загрузчик. Парсит PE-заголовки beacon DLL прямо из секции .co2pay, выделяет память через VirtualAlloc, маппит секции, фиксит базовые релокации (HIGHLOW / DIR64), резолвит импорты и вызывает DllEntry.

Ни один файл не записывается на диск. В файловой телеметрии (Sysmon Event 11, MFT) следов нет. Главный вектор обнаружения — сканирование памяти (RWX / unbacked executable pages).

Этапы работы:

1. VirtualAlloc(preferred_base, SizeOfImage, RW)
2. raw_copy(headers)
3. raw_copy(sections[])
4. apply_relocations(delta)
5. resolve_imports(LoadLibraryA + GetProcAddress)
6. VirtualProtect(sections[] -> RX / RW / R)
7. call DllEntry(DLL_PROCESS_ATTACH)

exe_inject

Remote process DLL injection. Записывает beacon DLL во временный файл, запускает целевой процесс (по умолчанию notepad.exe) скрыто, затем инжектирует DLL через классическую связку:

VirtualAllocEx(target, ..., RW)
WriteProcessMemory(target, dllpath)
CreateRemoteThread(target, LoadLibraryW, dllpath)

После инжекта стаб удаляет временный файл и завершается. Beacon работает внутри целевого процесса.

Замените TARGET в исходнике на нужный процесс (svchost.exe, RuntimeBroker.exe и т.д.).

exe_ppid

PPID-spoofing + injection. Находит explorer.exe, создаёт дочерний процесс (notepad.exe) с подменённым родителем через PROC_THREAD_ATTRIBUTE_PARENT_PROCESS, затем инжектирует beacon.

В дереве процессов beacon выглядит как легитимный потомок explorer.exe, а не как дочерний процесс стаба.

OpenProcess(explorer.exe, PROCESS_CREATE_PROCESS)
InitializeProcThreadAttributeList(1)
UpdateProcThreadAttribute(PARENT_PROCESS, &hExplorer)
CreateProcessW(notepad.exe, CREATE_SUSPENDED | EXTENDED_STARTUPINFO)
ResumeThread -> WaitForInputIdle -> inject
EDR-системы, проверяющие PID-цепочку (parent spoofing detection), могут сопоставить CreatorProcess (ETW) с заявленным ParentProcess и обнаружить подмену.

dll_sideload

DLL для side-loading (DLL search-order hijacking). Переименуйте выходной файл в имя DLL, которую приложение загружает без полного пути (например, version.dll, winmm.dll, dwrite.dll), и поместите рядом с EXE приложения.

Beacon загружается на отдельном потоке из DllMain (через CreateThread — чтобы избежать deadlock на loader lock).

Для проксирования экспортов реальной DLL добавьте forwarding stubs или .def-файл с директивами #pragma comment(linker, "/export:...").

Сборка (build_all.bat)

Скрипт build_all.bat компилирует каждый стаб для x64 и x86:

cd kit\artifact
build_all.bat

Результат: artifact\<name>.x64.exe / .x86.exe (или .dll для стабов с префиксом dll_).

Флаги компилятора

ФлагНазначение
/ZlНе записывать default-library запись в .obj
/GS-Отключить stack cookies (нет CRT __security_cookie)
/O2Оптимизация размера/скорости
/TCКомпилировать как C

Флаги линкера

ФлагНазначение
/NODEFAULTLIBНет CRT
/ENTRY:stub_mainТочка входа (или /ENTRY:DllMain для dll_*)
/SUBSYSTEM:WINDOWSGUI-подсистема (нет окна консоли)

Автоопределение типа

Стабы с префиксом dll_ автоматически собираются как DLL (с /DLL, /ENTRY:DllMain). Все остальные — как EXE с /ENTRY:stub_main.

Как написать свой стаб

  1. Создайте каталог kit/artifact/<name>/.
  2. Создайте файл <name>/<name>.c.
  3. Подключите общий заголовок: 
    #include "../artifact.h"
ART_DECLARE_PAYLOAD();
  4. Реализуйте точку входа: 
    void __stdcall stub_main(void) {
    unsigned int sz = art_get_size(g_payload);
    if (!sz) ExitProcess(1);

    // ... ваша логика загрузки ...

    ExitProcess(0);
}
    Для DLL-стабов (имя начинается с dll_): 
    BOOL WINAPI DllMain(HINSTANCE inst, DWORD reason, LPVOID reserved) {
    // ...
    return TRUE;
}
  5. Запустите build_all.bat — ваш стаб подхватится автоматически.

Ограничения (без CRT)

  • Нет memcpy / memset / strlen — используйте art_memcpy из заголовка или WinAPI.
  • Нет = { 0 } для больших массивов на стеке (может вызвать implicit memset) — используйте static или VirtualAlloc.
  • x86: с /Gs9999999 нет stack-probe. Локальные массивы > 4 КБ = crash. Выделяйте на куче (HeapAlloc).
  • Не используйте ZeroMemory / CopyMemory (это макросы на memset/memcpy).
  • Доступны только kernel32.lib и user32.lib. Другие библиотеки подгружайте через LoadLibraryW + GetProcAddress.
