Диагностика и типовые проблемы

Клиент — Authentication failed

  • Проверить operator.crt подписан тем же CA, что и server.crt.
  • Username/password совпадают с тем, что задано на teamserver'е.
  • Время на клиенте и сервере не разъезжается > 5 мин (TLS).

Beacon не появляется в Sessions

  • Listener реально слушает порт — проверить netstat -ano | findstr :443.
  • Ключ listener'а в beacon'е совпадает с тем, что отображается во вкладке Listeners. Если teamserver был перезапущен — ключ сменился, beacon нужно перегенерировать.
  • Файрвол / SmartScreen блокируют payload.exe. Проверить, что исполняемый файл не помещён в карантин антивирусом.

execute-assembly: hr=0x80131522

Сборка не подходит для CLR v4 (например .NET 6). Перекомпилировать под --targetframework net48 или net4.8.

execute-assembly: вывод пустой

Stdout сборки в текущем варианте не перехватывается. Используйте spawnto для запуска в дочернем процессе или модифицируйте сборку, чтобы она писала результат в файл/socket.

inject_apc — APCs queued: 0

В целевом процессе нет ни одного потока в alertable wait. Подойдут explorer.exe, svchost.exe с NetSvcs; не подойдёт notepad.exe в idle.

BOF падает

COFF собран не для x64 или содержит неподдерживаемые типы релокаций. Использовать x86_64-w64-mingw32-gcc -c или MSVC с /c /favor:AMD64. Проверить целевую архитектуру через dumpbin /headers.

Generate artifact — sentinel not found

Шаблон beacon'а собран без области для подстановки конфигурации. Использовать стандартный шаблон beacon.dll / beacon.exe из дистрибутива.
