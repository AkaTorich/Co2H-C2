Artifact Kit

Artifact Kit is a set of template executables (stubs) into which artifact-gen embeds the compiled beacon DLL. Each stub implements its own delivery/loading technique for the beacon on the target. All stubs are compiled without CRT (/NODEFAULTLIB, /Zl, /GS-).

Directory Structure

kit/artifact/
+-- artifact.h            -- shared header (macros, helpers)
+-- build_all.bat         -- build all stubs (x64 + x86)
+-- exe_basic/            -- basic EXE: temp file + LoadLibraryW
+-- exe_debug/            -- debug EXE: MessageBox at each stage
+-- exe_svc/              -- Windows Service
+-- exe_reflective/       -- In-memory PE loader (no disk write)
+-- exe_inject/           -- Remote DLL injection into another process
+-- exe_ppid/             -- PPID-spoofing + injection
+-- dll_sideload/         -- DLL for side-loading / search-order hijack
+-- artifact/             -- output directory (compiled .exe/.dll)

.co2pay Section

Each stub contains a named PE section .co2pay — a slot for the beacon DLL. Sentinel at compile time:

43 4F 32 48 50 41 59 4C  00 00 00 00 ...   ("CO2HPAYL" + 4-byte size + zeros)

artifact-gen locates this sentinel in the compiled stub, writes the DLL size into bytes [8..11] and copies the DLL body starting at offset 12. Maximum size: 512 KB.

Macro from artifact.h:

ART_DECLARE_PAYLOAD();
// Expands to:
//   #pragma section(".co2pay", read)
//   __declspec(allocate(".co2pay"))
//   static const unsigned char g_payload[512*1024 + 12] = {"CO2HPAYL", 0, ...};

Shared Header artifact.h

The file kit/artifact/artifact.h contains repeating code shared by all stubs:

Function / MacroPurpose
ART_DECLARE_PAYLOAD()Declare the .co2pay section with the sentinel
art_get_size(payload)Verify magic + return DLL size (0 on error)
art_memcpy(dst, src, n)Byte-by-byte copy without CRT
art_write_temp(path, payload, sz)Write DLL to a temp file (%TEMP%)
art_load_beacon(payload, sz)Write + LoadLibraryW + delete file
art_inject_remote(hProcess, path)Inject via CreateRemoteThread(LoadLibraryW)

All functions are declared static — each stub gets its own copy; unused ones are removed by the linker.

Stub Descriptions

exe_basic

Minimal EXE. Writes the DLL to a temp file, loads it via LoadLibraryW, deletes the file, sleeps forever. Entry point: stub_main. Use as a template for your own stubs.

exe_debug

Identical to exe_basic, but shows a MessageBoxA at each stage and installs a Vectored Exception Handler (VEH) to catch exceptions inside the beacon's DllEntry. Use when debugging load issues.

Do not use in operations — dialogs are visible to the user.

exe_svc

Windows Service wrapper. When started via SCM it registers a ServiceCtrlHandler, reports SERVICE_RUNNING and keeps the beacon alive. When run directly (not via SCM) it silently falls back to exe_basic behavior.

sc create BeaconSvc binPath= "C:\path\exe_svc.x64.exe" start= auto
sc start  BeaconSvc
Service API (advapi32.dll) is loaded dynamically — the stub links only with kernel32.lib + user32.lib.

exe_reflective

In-memory loader. Parses the beacon DLL PE headers directly from the .co2pay section, allocates memory via VirtualAlloc, maps sections, fixes base relocations (HIGHLOW / DIR64), resolves imports and calls DllEntry.

No file is written to disk. No traces in file telemetry (Sysmon Event 11, MFT). The primary detection vector is memory scanning (RWX / unbacked executable pages).

Execution stages:

1. VirtualAlloc(preferred_base, SizeOfImage, RW)
2. raw_copy(headers)
3. raw_copy(sections[])
4. apply_relocations(delta)
5. resolve_imports(LoadLibraryA + GetProcAddress)
6. VirtualProtect(sections[] -> RX / RW / R)
7. call DllEntry(DLL_PROCESS_ATTACH)

exe_inject

Remote process DLL injection. Writes the beacon DLL to a temp file, launches the target process (default notepad.exe) hidden, then injects the DLL via the classic combination:

VirtualAllocEx(target, ..., RW)
WriteProcessMemory(target, dllpath)
CreateRemoteThread(target, LoadLibraryW, dllpath)

After injection the stub deletes the temp file and exits. The beacon runs inside the target process.

Replace TARGET in the source with the desired process (svchost.exe, RuntimeBroker.exe, etc.).

exe_ppid

PPID-spoofing + injection. Finds explorer.exe, creates a child process (notepad.exe) with a spoofed parent via PROC_THREAD_ATTRIBUTE_PARENT_PROCESS, then injects the beacon.

In the process tree, the beacon appears as a legitimate child of explorer.exe, not as a child of the stub.

OpenProcess(explorer.exe, PROCESS_CREATE_PROCESS)
InitializeProcThreadAttributeList(1)
UpdateProcThreadAttribute(PARENT_PROCESS, &hExplorer)
CreateProcessW(notepad.exe, CREATE_SUSPENDED | EXTENDED_STARTUPINFO)
ResumeThread -> WaitForInputIdle -> inject
EDR systems that check the PID chain (parent spoofing detection) may correlate CreatorProcess (ETW) with the claimed ParentProcess and detect the spoof.

dll_sideload

DLL for side-loading (DLL search-order hijacking). Rename the output file to the name of the DLL that an application loads without a full path (e.g. version.dll, winmm.dll, dwrite.dll), and place it alongside the application EXE.

The beacon is loaded on a separate thread from DllMain (via CreateThread — to avoid deadlock on the loader lock).

To forward exports of the real DLL, add forwarding stubs or a .def file with #pragma comment(linker, "/export:...") directives.

Building (build_all.bat)

The script build_all.bat compiles each stub for x64 and x86:

cd kit\artifact
build_all.bat

Output: artifact\<name>.x64.exe / .x86.exe (or .dll for stubs with the dll_ prefix).

Compiler Flags

FlagPurpose
/ZlDo not write a default-library record into .obj
/GS-Disable stack cookies (no CRT __security_cookie)
/O2Size/speed optimization
/TCCompile as C

Linker Flags

FlagPurpose
/NODEFAULTLIBNo CRT
/ENTRY:stub_mainEntry point (or /ENTRY:DllMain for dll_*)
/SUBSYSTEM:WINDOWSGUI subsystem (no console window)

Automatic Type Detection

Stubs with the dll_ prefix are automatically built as DLLs (with /DLL, /ENTRY:DllMain). All others — as EXE with /ENTRY:stub_main.

How to Write Your Own Stub

  1. Create a directory kit/artifact/<name>/.
  2. Create the file <name>/<name>.c.
  3. Include the shared header: 
    #include "../artifact.h"
ART_DECLARE_PAYLOAD();
  4. Implement the entry point: 
    void __stdcall stub_main(void) {
    unsigned int sz = art_get_size(g_payload);
    if (!sz) ExitProcess(1);

    // ... your loading logic ...

    ExitProcess(0);
}
    For DLL stubs (name starts with dll_): 
    BOOL WINAPI DllMain(HINSTANCE inst, DWORD reason, LPVOID reserved) {
    // ...
    return TRUE;
}
  5. Run build_all.bat — your stub will be picked up automatically.

Restrictions (no CRT)

  • No memcpy / memset / strlen — use art_memcpy from the header or WinAPI.
  • No = { 0 } for large stack arrays (may trigger implicit memset) — use static or VirtualAlloc.
  • x86: with /Gs9999999 there is no stack probe. Local arrays > 4 KB = crash. Allocate on the heap (HeapAlloc).
  • Do not use ZeroMemory / CopyMemory (these are macros over memset/memcpy).
  • Only kernel32.lib and user32.lib are available. Load other libraries via LoadLibraryW + GetProcAddress.
