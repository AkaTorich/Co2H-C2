Quick Start

1. mTLS Certificates

Generated automatically on the first project build, or manually via the Tools → Generate Certificates menu. Files appear in the certs\ directory: ca.crt, server.crt/server.key, operator.crt/operator.key.

2. Starting the Client

Double-click co2h-client.exe. In the login dialog:

FieldDefault Value
Host127.0.0.1
Port5050
Usernameadmin
Passwordchangeme
CAcerts\ca.crt
Cert / Keycerts\operator.crt / certs\operator.key

3. Starting the Teamserver

On the client toolbar — click Start Server. Starts the built-in teamserver alongside the client, uses configs/srv.toml. Stdout/stderr of the process go to the Logs tab. Stop Server shuts it down.

4. Creating a Listener

An HTTPS listener starts automatically from srv.toml. TCP, SMB and DNS are created via the client: toolbar → New listener (or the Add button in the Listeners tab), select type, enter parameters. Once created, the listener appears in the list with its key.

For DNS: specify the C2 domain — the subdomain for which the teamserver is the authoritative NS server (NS record at the registrar).

5. Generating a Beacon

Toolbar → Generate artifact. In the Listener dropdown, select the desired listener — the key and parameters will be filled in automatically. Specify the format (EXE/DLL), sleep/jitter and save path.

The beacon embeds the address and key of the selected listener. On startup it connects to that listener; the teamserver identifies the beacon by the encrypted check-in.

6. Running the Beacon on the Target

Copy payload.exe to the test machine (Windows 10/11 x64) and run it. After a few seconds the beacon will appear in the Sessions tab.

7. Interacting

Double-click a beacon row → the bottom console becomes active. Type pwd and press Enter.
