Troubleshooting and Common Issues

Client — Authentication failed

  • Verify that operator.crt is signed by the same CA as server.crt.
  • Username/password match what is configured on the teamserver.
  • Time on client and server does not diverge by more than 5 minutes (TLS).

Beacon not appearing in Sessions

  • Listener is actually listening on the port — check netstat -ano | findstr :443.
  • The listener key in the beacon matches what is shown in the Listeners tab. If the teamserver was restarted — the key changed, the beacon needs to be regenerated.
  • Firewall / SmartScreen is blocking payload.exe. Verify that the executable has not been quarantined by antivirus.

execute-assembly: hr=0x80131522

The assembly is not compatible with CLR v4 (e.g. .NET 6). Recompile targeting --targetframework net48 or net4.8.

execute-assembly: empty output

Stdout of the assembly is not captured in the current implementation. Use spawnto to run in a child process, or modify the assembly to write its output to a file or socket.

inject_apc — APCs queued: 0

The target process has no threads in an alertable wait state. Try explorer.exe, svchost.exe with NetSvcs; notepad.exe in idle will not work.

BOF crashes

The COFF was not compiled for x64, or contains unsupported relocation types. Use x86_64-w64-mingw32-gcc -c or MSVC with /c /favor:AMD64. Check the target architecture with dumpbin /headers.

Generate artifact — sentinel not found

The beacon template was built without the configuration placeholder area. Use the standard beacon.dll / beacon.exe template from the distribution package.
