Plugin API

The Co2H client supports dynamic plugins — shared libraries (.dll on Windows, .so on Linux) that are loaded from the plugins/ folder alongside the co2h-client executable at startup.

A plugin can:

  • Add tabs to the main window
  • Register console commands (with auto-completion and hints)
  • Send tasks to beacons and receive results via callbacks
  • Write to the console and the Logs tab
  • Read data models (sessions, listeners, credentials, downloads)

Plugin Structure

Each plugin is a separate DLL/SO exporting two C functions:

#include "client/plugin/IClientPlugin.hpp"

CO2H_PLUGIN_EXPORT
co2h::client::plugin::IClientPlugin* co2h_plugin_create();

CO2H_PLUGIN_EXPORT
void co2h_plugin_destroy(co2h::client::plugin::IClientPlugin*);

The CO2H_PLUGIN_EXPORT macro expands to extern "C" __declspec(dllexport) on Windows and extern "C" __attribute__((visibility("default"))) on Linux.

IClientPlugin Interface

namespace co2h::client::plugin {

struct PluginInfo {
    const char* name;        // short name ("my_recon")
    const char* version;     // "1.0.0"
    const char* author;
    const char* description; // one line
};

class IClientPlugin {
public:
    virtual ~IClientPlugin() = default;

    // Plugin metadata.
    virtual PluginInfo info() const = 0;

    // Called once after loading.
    // ctx — PluginContext object for interacting with the client.
    virtual void initialize(PluginContext* ctx) = 0;

    // Called before unloading. Release resources.
    virtual void shutdown() = 0;
};

}

PluginContext — API for Plugins

Plugin Widget Management

MethodDescription
int addPluginButton(QWidget* w, const QIcon& icon, const QString& label) Add a plugin widget. An icon button appears on the plugin panel. The first click on the button opens the plugin interface (main tabs are hidden), a second click on the same button hides the plugin and restores the client tabs. Returns the index. The widget belongs to the plugin — it must be deleted in shutdown().
void removePluginButton(QWidget* w) Remove a previously added plugin widget.

Executing Tasks

Named methods for common operations — the plugin does not need to know numeric protocol codes:

MethodDescription
shell(beaconId, cmd, cb) Execute a command via cmd.exe /c.
run(beaconId, cmdline, cb) Launch a process directly (without cmd.exe).
ps(beaconId, cb) Process list.
ls(beaconId, path, cb) Directory listing (empty path = current).
pwd(beaconId, cb) Current directory.
getuid(beaconId, cb) Current user / SID.
screenshot(beaconId, cb) Screen capture.
download(beaconId, remotePath, cb) Download a file from the target.
upload(beaconId, payload, cb) Upload data to the target.

Each method returns an rpc_id (0 on error). cb — optional callback void(const QByteArray& output, const QString& error), called when the result is received.

Routing is fully automatic: the rpc_id is bound to the callback; when the server responds, the binding is transferred to the task_id; when output arrives — the callback is called and removed.

For non-standard operations (BOF, custom extensions) a low-level sendTask(beaconId, op, payload, cb) method is available, accepting a numeric operation code.

Executing Built-in Commands

MethodDescription
execCommand(beaconId, command, cb) Execute any built-in client command by name and receive output in the callback. The command string is the full command as typed in the console, e.g. "bof enum.o go -- z domain.local" or "hashdump". Can be called multiple times in a row — each callback receives its own result.

Usage example:

ctx_->execCommand(bid, "hashdump",
    [this](const QByteArray& out, const QString& err) {
        if (err.isEmpty())
            parseHashes(QString::fromUtf8(out));
        else
            ctx_->consoleError("[myplugin] hashdump failed: " + err);
    });

Console Commands

MethodDescription
void registerCommand(name, argsHelp, description, handler) Register a command. handler — function void(const QString& beaconId, const QString& args). The command appears in the console auto-completion with a [plugin] tag.

Output

MethodDescription
void consoleWrite(const QString& text) Write text to the active beacon's console (light color).
void consoleError(const QString& text) Write an error to the console (red color).
void log(const QString& text) Write a line to the Logs tab.

Access to Data Models

MethodTypeDescription
serverClient()ServerClient* Low-level teamserver client. For advanced use cases.
sessionsModel()SessionsModel* Beacon model. row(i) returns a BeaconRow* (id, host, user, pid, arch, ip, listener, etc.).
listenersModel()ListenersModel* Listener model.
downloadsModel()DownloadsModel* Downloads model.
credentialsModel()CredentialsModel* Credentials model.

Example: "Recon" Plugin

A minimal plugin that adds a Recon tab and a recon console command. When invoked, it runs three reconnaissance commands on the beacon and collects results in its own tab.

#include "client/plugin/IClientPlugin.hpp"
#include "client/plugin/PluginContext.hpp"

#include <QTextEdit>
#include <QVBoxLayout>
#include <QWidget>

using namespace co2h::client;
using namespace co2h::client::plugin;

class ReconPlugin : public IClientPlugin {
public:
    PluginInfo info() const override {
        return {"recon", "1.0.0", "operator",
                "Basic recon: whoami + ipconfig + net user"};
    }

    void initialize(PluginContext* ctx) override {
        ctx_ = ctx;

        // Create the tab widget.
        tab_ = new QWidget;
        out_ = new QTextEdit(tab_);
        out_->setReadOnly(true);
        auto* lay = new QVBoxLayout(tab_);
        lay->addWidget(out_);

        ctx_->addPluginButton(tab_, QIcon(), "Recon");

        // Register the command.
        ctx_->registerCommand(
            "recon", "[target]",
            "whoami + ipconfig + net user on the beacon",
            [this](const QString& bid, const QString&) {
                runRecon(bid);
            });
    }

    void shutdown() override {
        if (ctx_ && tab_) ctx_->removePluginButton(tab_);
        delete tab_; tab_ = nullptr;
    }

private:
    void runRecon(const QString& bid) {
        ctx_->consoleWrite("[recon] starting...");

        ctx_->shell(bid, "whoami /all",
            [this](const QByteArray& o, const QString& e) {
                if (e.isEmpty())
                    out_->append("=== whoami ===\n"
                        + QString::fromLocal8Bit(o));
                else
                    out_->append("ERROR: " + e);
            });

        ctx_->shell(bid, "ipconfig /all",
            [this](const QByteArray& o, const QString& e) {
                if (e.isEmpty())
                    out_->append("=== ipconfig ===\n"
                        + QString::fromLocal8Bit(o));
            });
    }

    PluginContext* ctx_ = nullptr;
    QWidget*       tab_ = nullptr;
    QTextEdit*     out_ = nullptr;
};

// Exported functions.
CO2H_PLUGIN_EXPORT IClientPlugin* co2h_plugin_create() {
    return new ReconPlugin;
}
CO2H_PLUGIN_EXPORT void co2h_plugin_destroy(IClientPlugin* p) {
    delete p;
}

Plugin Icon

Each plugin stores its icon inside the source code as a string constant containing the full SVG file text. The icon does not depend on client resources — when loaded, the plugin renders the SVG into a QIcon itself.

Icon Embedding Template

#include <QIcon>
#include <QPainter>
#include <QPixmap>
#include <QSvgRenderer>

// Full SVG text — stored as a raw string literal.
static const char kMyPluginSvg[] = R"SVG(
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24">
  <!-- ... icon content ... -->
</svg>
)SVG";

// Helper function: SVG string -> QIcon.
static QIcon iconFromSvg(const char* data) {
    QByteArray svg(data);
    QSvgRenderer r(svg);
    QPixmap pix(24, 24);
    pix.fill(Qt::transparent);
    QPainter p(&pix);
    r.render(&p);
    return QIcon(pix);
}

Usage When Registering

void initialize(PluginContext* ctx) override {
    // ...
    ctx->addPluginButton(tab_, iconFromSvg(kMyPluginSvg), "My Plugin");
}
SVG requirements:
  • viewBox size — 0 0 24 24 (standard for client icons).
  • Gradients, opacity, any SVG primitives are allowed.
  • Do not use external references (xlink:href to external files).
  • Rendering requires the Qt6::Svg module — add it to CMakeLists.txt (see below).

Building a Plugin

Minimal CMakeLists.txt for a plugin:

cmake_minimum_required(VERSION 3.24)
project(my_plugin LANGUAGES CXX)

find_package(Qt6 6.4 COMPONENTS Core Gui Widgets Svg REQUIRED)
set(CMAKE_AUTOMOC ON)

add_library(my_plugin SHARED MyPlugin.cpp)
target_link_libraries(my_plugin PRIVATE
    Qt6::Core Qt6::Gui Qt6::Widgets Qt6::Svg)

set_target_properties(my_plugin PROPERTIES
    OUTPUT_NAME "my_plugin"
    PREFIX ""
    RUNTIME_OUTPUT_DIRECTORY "${CMAKE_BINARY_DIR}/plugins"
)
The plugin must be compiled with the same compiler and Qt version as the client. A mismatch will cause a crash on load.

Lifecycle

  1. On startup, the client scans plugins/ for *.dll (Windows) or *.so (Linux).
  2. For each file, LoadLibrary / dlopen is called.
  3. The exports co2h_plugin_create and co2h_plugin_destroy are looked up. If at least one is not found — the file is skipped.
  4. co2h_plugin_create() is called → pointer to IClientPlugin.
  5. initialize(ctx) is called — the plugin adds tabs, commands, etc.
  6. On client exit: shutdown() → co2h_plugin_destroy(ptr) → FreeLibrary / dlclose.

The number of loaded plugins is logged to the Logs tab on startup.

Tips

  • Delete your widgets in shutdown(), not in the destructor — by the time the destructor is called, QTabWidget may already be destroyed.
  • The callback from sendTask is called once. For repeating tasks, send a new sendTask from the callback.
  • Do not block the UI thread with long-running operations — use QThread or QtConcurrent.
  • Models (sessionsModel(), etc.) are read-only. Writing through them is not prohibited but is not recommended.
  • To send tasks, use numeric opcodes from beacon.h. New opcodes can be defined alongside custom BOFs.
Beacon output encoding. The beacon sends text in UTF-8 on Windows and in the system code page on Linux. When decoding QByteArray from a callback, use conditional compilation: 
#ifdef _WIN32
    QString text = QString::fromUtf8(output);
#else
    QString text = QString::fromLocal8Bit(output);
#endif
Without this, non-ASCII characters will display incorrectly.
